1. Library Dependency Graph

The fork creates two new libraries under ydb/core/kqp/opt/cbo/. The key invariant: the fork depends on shared DQ infrastructure (dq/opt/core) but never on the YQL-owned CBO code (yql/essentials/core/cbo or ydb/library/yql/dq/opt).

Why this split?

The original ydb/library/yql/dq/opt is a single library containing both CBO solver code and shared DQ infrastructure (dq_opt.h, dq_opt_phy.h, etc.). To fork the CBO without duplicating infrastructure, we split dq/opt into:

  • dq/opt/core — infrastructure only (uses SRCDIR to compile files from the parent dir without moving them)
  • dq/opt — CBO solver (now depends on core)

The fork's solver/ depends on dq/opt/core for infrastructure but compiles its own copies of the CBO solver files.

2. File Map & Classification

Every file in the fork falls into one of three tiers:

  • CBO — Core optimizer logic: algorithms, types, cost model, hypergraph
  • STATS — Statistics infrastructure: stat inference, propagation, selectivity. Not CBO per se but tightly coupled to CBO types
  • PLUMB — Join rewrite plumbing: included because CBO type changes propagate through function signatures

The fork introduces a clean boundary between NYql types (shared, read-only) and NKikimr::NKqp types (owned by KQP). Here is exactly where conversions occur:

Boundary Point Details

kqp_statistics_transformer.cpp — FromYqlColumnStatMap() (file-scope static)

Iterates THashMap<TString, NYql::TColumnStatistics> and field-by-field copies each entry into NKikimr::NKqp::TColumnStatistics. Both structs have identical fields (NumUniqueVals, HyperLogLog, CountMinSketch, EqWidthHistogramEstimator, Type); shared pointers are copied as-is. Lives in the transformer rather than kqp_statistics.cpp because the argument type requires #include <yql/essentials/core/yql_statistics.h>, which transitively includes yql/essentials/core/cbo — a dependency that kqp/opt/cbo explicitly denies via CHECK_DEPENDENT_DIRS.

kqp_statistics.cpp — TKqpStatsStore::GetStats() / SetStats()

Keyed by NYql::TExprNode::UniqueId() (a ui64). Stores NKikimr::NKqp::TOptimizerStatistics. This is the KQP parallel to TTypeAnnotationContext::StatisticsMap — decoupled so KQP optimizer passes own their stats without mutating the shared YQL context.

dq_opt_join.cpp:182-189 — EJoinAlgoType static_cast

TEquiJoinLinkSettings::JoinAlgo is NYql::EJoinAlgoType (from the YQL equi-join infrastructure). TOptimizerHints::TJoinAlgoHint::Algo is NKikimr::NKqp::EJoinAlgoType (from the fork). Both enums have identical integer values by design. The cast occurs in two directions: hint → link settings, and link settings → local variable for KQP enum comparisons.

kqp_statistics_transformer.cpp:1012-1017 — FSM wiring

Builds NKikimr::NKqp::TOrderingsStateMachine instances (shufflings + sortings) from collected FDs and stores them on TKqpStatsStore.ShufflingsFSM / .SortingsFSM. The YQL TTypeAnnotationContext also gets its own FSMs built from the same data, but those use NYql::NDq::TOrderingsStateMachine. The two FSM types are structurally identical but live in different namespaces.

4. Substantive Change Rationale

For each file with non-mechanical diffs, here is why the change was necessary — expressed as dependency chains.

kqp_statistics.h (+80/-34)

Completely restructured. Root cause: TOptimizerStatistics in yql/essentials/core/yql_statistics.h uses NYql::NDq::TJoinColumn, NYql::NDq::TOrderingsStateMachine::TLogicalOrderings, and NYql::TColumnStatistics. If we just included that header, every CBO type in the fork would be an NYql type, defeating the purpose.

→ So we duplicated TOptimizerStatistics and all its nested types into NKikimr::NKqp, using the fork's own TJoinColumn and TOrderingsStateMachine. Added TKqpStatsStore as a KQP-owned stats map (parallel to TTypeAnnotationContext::StatisticsMap). The one boundary conversion (FromYqlColumnStatMap) lives in kqp_statistics_transformer.cpp rather than here because including yql_statistics.h in the kqp/opt/cbo library would transitively pull in yql/essentials/core/cbo, violating the CHECK_DEPENDENT_DIRS restriction.

cbo_optimizer_new.h (+45/-37)

Root cause: The original defines NYql::EJoinAlgoType (which lives in yql/essentials/core/yql_cost_function.h, not even in this file). The fork's copy needs its own NKikimr::NKqp::EJoinAlgoType so that all CBO types (TJoinAlgoHints, IProviderContext) use the KQP enum.

→ Added the EJoinAlgoType enum + AllJoinAlgos constexpr directly. Removed #include <yql_cost_function.h>. Changed all NDq::TJoinColumn references to the fork's TJoinColumn. Forward-declared NYql::TExprContext (needed by IOptimizerFactory).

cbo_interesting_orderings.h (+38/-6)

Root cause: The original #include <yql_cost_function.h> pulls in NYql::NDq::TJoinColumn. But the fork needs its own TJoinColumn in NKikimr::NKqp to avoid using YQL types.

→ Inlined the entire TJoinColumn struct (with operator==, operator<, THashFunction implementations in the .cpp). Removed TShuffling::SetNatural() and IsNatural field (unused after orderings rework). Added GetColumns() accessor.

dq_opt_stat.h (+27/-23) and dq_opt_stat.cpp (+235/-237)

Root cause: Every InferStatisticsFor*() function takes TTypeAnnotationContext* and calls typeCtx->GetStats() / typeCtx->SetStats(). But TTypeAnnotationContext is shared YQL state. The fork needs its own stats store.

→ Replaced TTypeAnnotationContext* with TKqpStatsStore* in all 20+ function signatures. Changed all typeCtx->GetStats/SetStats to kqpStats->GetStats/SetStats. For functions that also need typeCtx for non-stats purposes (e.g., InferStatisticsForExtendBase accesses OrderingsFSM), added both parameters. Functions like NeedCalc, IsConstantExpr, IsLiteralDataExpr were moved into an anonymous namespace and reordered due to forward-declaration requirements.

dq_opt_stat_transformer_base.h (+5/-2) and .cpp (+33/-27)

Root cause: TDqStatisticsTransformerBase stores TypeCtx and passes it to all InferStatisticsFor*() calls. After the stat function signatures changed, the transformer also needs TKqpStatsStore*.

→ Added TKqpStatsStore* KqpStats member. Constructor takes it as a new parameter. All InferStatisticsFor*(input, TypeCtx) calls changed to InferStatisticsFor*(input, KqpStats) (or InferStatisticsFor*(input, KqpStats, TypeCtx) for the few that need both).

dq_opt_join.h (+13/-18) and dq_opt_join.cpp (+15/-7)

Root cause: DqRewriteEquiJoin() calls typeCtx.GetStats()/typeCtx.SetStats() to propagate equi-join statistics. It also processes TOptimizerHints which use the fork's EJoinAlgoType.

→ Added TKqpStatsStore& parameter to both DqRewriteEquiJoin overloads. Introduced static_cast between NYql::EJoinAlgoType (in TEquiJoinLinkSettings) and NKikimr::NKqp::EJoinAlgoType (in hints). Removed DqCollectJoinRelationsWithStats declaration (moved to dq_opt_join_cost_based.h where it uses TKqpStatsStore).

dq_opt_join_cost_based.h (+11/-0) and .cpp (+15/-5)

Root cause: DqOptimizeEquiJoinWithCosts() is the main CBO entry point. It calls typesCtx.GetStats() to collect relation stats and typesCtx.SetStats() to store results.

→ Added TKqpStatsStore& parameter to all overloads. DqCollectJoinRelationsWithStats moved here (was in dq_opt_join.h) with TKqpStatsStore parameter. All typesCtx.GetStats/SetStats calls replaced with kqpStats.GetStats/SetStats.

dq_opt_join_cbo_factory.cpp (+2/-3)

Root cause: MakeJoinCostBasedOptimizerPG calls MakePgOptimizerNew() from yql/essentials/parser/pg_wrapper. The fork doesn't need/want the PG optimizer dependency.

→ Replaced PG path with Y_ABORT("PG optimizer is not supported in the ydb-owned CBO"). Removed pg_wrapper/interface/optimizer.h include.

dq_opt_join_hypergraph.h (+0/-1)

Root cause: Original includes yql_cost_function.h (for TJoinColumn). The fork's TJoinColumn lives in cbo_interesting_orderings.h instead.

→ Removed the one include line. TJoinColumn is already available via cbo_optimizer_new.h → cbo_interesting_orderings.h.

dq_opt_predicate_selectivity.cpp (+3/-1)

Root cause: TPredicateSelectivityComputer methods were defined with explicit NYql::NDq:: qualification. In the fork they live in NKikimr::NKqp.

→ Wrapped implementations in namespace NKikimr::NKqp { ... }. Removed explicit NYql::NDq:: qualification on method definitions.

dq_cbo_ut.cpp (+4/-16)

Root cause: Tests use TTypeAnnotationContext as stats storage and call MakePgOptimizerNew for the PG test.

→ Replaced TTypeAnnotationContext typeCtx with TKqpStatsStore kqpStats in all test functions. Removed the DqOptimizeEquiJoinWithCostsPG test (PG not supported). Removed pg_wrapper include. Suite renamed from DQCBO to KqpCBO.

dq_opt_hypergraph_ut.cpp (+1/-3)

Root cause: Tests reference NDq::TJoinColumn and use NYql::NDq namespace.

→ Consolidated three using namespace lines into one using namespace NKikimr::NKqp. Changed include to the fork's dq_opt_join.h. Suite renamed to KqpHypergraphBuild.

5. Why Each Non-CBO File Is Copied

The dependency chain problem

Even files that aren't "CBO" in the algorithmic sense must be copied because of transitive type dependencies:

dq_opt_join.h/cpp (Join rewrite plumbing)

DqRewriteEquiJoin() calls GetStats()/SetStats() on the stats store. If we kept using the original, it would call typeCtx.GetStats() — the shared YQL stats map. We need it to use kqpStats.GetStats() instead. The function also processes TOptimizerHints which contains NKqp::EJoinAlgoType, requiring static_cast at the boundary with TEquiJoinLinkSettings::JoinAlgo (NYql::EJoinAlgoType).

dq_opt_join_tree_node.h/cpp

Defines ToString(EJoinAlgoType) and TJoinOptimizerNodeInternal which references the fork's EJoinAlgoType, EJoinKind, and TJoinColumn. If it used the originals, these types would be NYql::* and wouldn't match the fork's CBO interfaces.

dq_opt_conflict_rules_collector.h/cpp

Uses EJoinKind for commutativity/associativity predicates. These predicates are called by MakeJoinHypergraph() which passes the fork's NKqp::EJoinKind values. A mismatch between enum types would cause compilation errors.

dq_opt_stat.h/cpp (Statistics infrastructure)

Not CBO per se (doesn't decide join order), but provides the statistics data that CBO algorithms consume. Every stat inference function takes TKqpStatsStore* and produces NKqp::TOptimizerStatistics. Can't use the original because it produces NYql::TOptimizerStatistics and stores in TTypeAnnotationContext.

bitset.h, dq_opt_log.h

No type dependencies at all — purely mechanical namespace change. Copied for organizational coherence: the solver directory is self-contained, all its headers live together. dq_opt_log.h is even IDENTICAL after normalization.

7. Entry Points

The KQP optimizer calls into the fork at exactly two function entry points, plus the statistics transformer:

kqp_opt_log.cpp:215 — DqOptimizeEquiJoinWithCosts()

The CBO path. Called when useCBO is enabled for the query. Passes KqpCtx.KqpStats (the KQP-owned stats store), constructs an IOptimizerNew via MakeNativeOptimizerNew(), and gets back a rewritten equi-join tree.

kqp_opt_log.cpp:230 — DqRewriteEquiJoin()

The non-CBO path (rule-based join rewriting). Also uses KqpCtx.KqpStats for statistics propagation even when CBO is off.

ydb/core/kqp/opt/rbo/rules/apply_cbo.cpp — TOptimizeCBOTreeRule::SimpleMatchAndApply()

The RBO → CBO bridge. When the rule-based optimizer produces a TOpCBOTree operator, this rule fires. It converts the RBO join tree into CBO types (TRelOptimizerNode, TJoinOptimizerNode), calls MakeNativeOptimizerNew() to get an IOptimizerNew, runs JoinSearch(), then converts the optimized tree back to RBO operators via ConvertOptimizedTree(). Distinct from the DQ logical rewrite path in kqp_opt_log.cpp.

kqp_statistics_transformer.cpp — InferStatisticsFor*()

The statistics transformer walks the plan bottom-up and calls InferStatisticsFor*() for each operator. It passes KqpStats to all inference functions. This is where table-level stats (from TTypeAnnotationContext) enter the KQP stats store via FromYqlColumnStatMap().
