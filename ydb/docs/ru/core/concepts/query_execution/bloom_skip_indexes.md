# Bloom skip индексы и фильтрация

Bloom skip индексы — локальные вероятностные фильтры по данным колонок, которые позволяют на этапе чтения из хранилища отбрасывать фрагменты таблицы, где искомые значения с высокой вероятностью отсутствуют. Это уменьшает объём фактически прочитанных данных при селективных запросах.

## Отличие от глобальных вторичных и полнотекстовых индексов

* Глобальный [вторичный индекс](../glossary.md#secondary-index) — отдельная индексная структура с собственным ключом для доступа к данным.
* [Полнотекстовый индекс](../../dev/fulltext-indexes.md) — отдельный инвертированный индекс по тексту для поиска и ранжирования.
* Bloom skip хранится локально вместе с данными таблицы и работает как фильтр чтения: он не материализует отдельную индексную таблицу, а помогает [оптимизатору запросов](optimizer.md) и слою хранения отбрасывать нерелевантные фрагменты данных.

## Когда применять

* `bloom_filter` — когда в запросах часто есть фильтрация по конкретным значениям индексированной колонки (равенство, набор значений).
* `bloom_ngram_filter` — для строковых колонок, когда нужен n-граммный вариант Bloom skip фильтрации.

Подробнее о типах, ограничениях и примерах DDL см. [Bloom skip индексы](../../dev/bloom-skip-indexes.md) и [справочник YQL](../../yql/reference/syntax/alter_table/indexes.md#local-bloom).
