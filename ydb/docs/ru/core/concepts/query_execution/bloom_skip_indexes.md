# Bloom skip индексы и фильтрация

Bloom skip индексы — локальные вероятностные фильтры по данным колонок, которые позволяют на этапе чтения из хранилища отбрасывать фрагменты таблицы, где искомые значения с высокой вероятностью отсутствуют. Это уменьшает объём фактически прочитанных данных при селективных запросах.

## Отличие от глобальных вторичных и полнотекстовых индексов

* Глобальный [вторичный индекс](../glossary.md#secondary-index) — отдельная структура с собственным ключом; доступ к данным через индекс обычно задаётся явно (`VIEW` для строковых таблиц в YQL).
* [Полнотекстовый индекс](../../dev/fulltext-indexes.md) — отдельный инвертированный индекс по тексту; поиск идёт через `VIEW` и функции вроде `FulltextMatch`.
* Bloom skip хранится локально вместе с данными таблицы, не дублирует строки как отдельная индексная таблица и не требует `VIEW`: решение об использовании принимает [оптимизатор запросов](optimizer.md) при разборе предикатов.

## Когда применять

* `bloom_filter` — когда в запросах часто есть фильтрация по конкретным значениям индексированной колонки (равенство, набор значений).
* `bloom_ngram_filter` — для строковых колонок, когда важны предикаты по подстроке (в том числе `LIKE` / `ILIKE`), и нужно уменьшить чтение лишних фрагментов данных.

Подробнее о типах, ограничениях и примерах DDL см. [Bloom skip индексы](../../dev/bloom-skip-indexes.md) и [справочник YQL](../../yql/reference/syntax/alter_table/indexes.md#local-bloom).
