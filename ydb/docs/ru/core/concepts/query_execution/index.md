# Выполнение запросов

{{ ydb-short-name }} предоставляет единый интерфейс для выполнения запросов на языке [YQL](../../yql/reference/index.md) и единый распределённый движок их выполнения. Один и тот же синтаксис и одни и те же механизмы позволяют обращаться к данным разной природы: от строк в таблицах базы данных до неограниченных потоков событий в топиках и данных во внешних системах.

С точки зрения пользователя запросы выполняются к трём типам сущностей:

- [таблицы базы данных](#tables) — транзакционные (OLTP) и аналитические (OLAP) запросы к данным, хранящимся в {{ ydb-short-name }};
- [внешние источники данных](#federated) — федеративные запросы к данным, расположенным за пределами {{ ydb-short-name }};
- [топики](#streaming) — запросы к неограниченным потокам данных: потоковые запросы и бесконечные запросы к топикам.

Общий порядок обработки запроса, а также базовые понятия — сессии, транзакции, повторные выполнения, язык запросов и результирующие наборы — описаны в разделе [{#T}](execution_process.md).

## Запросы к таблицам базы данных {#tables}

Основной сценарий — выполнение запросов к [таблицам](../datamodel/table.md), хранящимся в {{ ydb-short-name }}. Единый интерфейс способен эффективно обрабатывать широкий спектр нагрузок — от высоконагруженных [транзакционных OLTP-запросов](https://ru.wikipedia.org/wiki/OLTP) до сложных [аналитических OLAP-запросов](https://ru.wikipedia.org/wiki/OLAP).

## Запросы к внешним источникам данных (федеративные запросы) {#federated}

[Федеративные запросы](federated_query/index.md) позволяют обращаться к данным, расположенным во внешних системах, без переноса этих данных внутрь {{ ydb-full-name }}. С помощью YQL-запросов можно читать данные из внешних СУБД и объектных хранилищ (S3), а также объединять их с данными в таблицах {{ ydb-short-name }}.

Подробнее см. в разделе [{#T}](federated_query/index.md).

## Запросы к топикам {#streaming}

Запросы к [топикам](../datamodel/topic.md) позволяют обрабатывать неограниченные потоки данных. Поскольку поток данных бесконечен, такой запрос не завершается после получения результата, а работает до явной отмены. Есть два варианта таких запросов.

### Потоковые запросы

[Потоковые запросы](../streaming-query.md) — основной способ потоковой обработки данных в production. Они оформляются как постоянные объекты схемы (`STREAMING QUERY`), читают сообщения из топиков по мере их поступления, записывают результаты в выходные топики или таблицы и автоматически восстанавливаются после сбоев из [чекпоинтов](../../dev/streaming-query/checkpoints.md).

Подробнее см. в разделе [{#T}](../streaming-query.md) и в описании [{#T}](../../yql/reference/syntax/create-streaming-query.md).

### Запросы к топикам в табличном режиме

Прочитать данные из топика можно и обычным `SELECT`, указав `STREAMING = "TRUE"` в блоке `WITH`. Без ограничения `LIMIT` такой запрос работает бесконечно, возвращая результат клиенту по мере поступления сообщений. В отличие от потоковых запросов, он не создаёт постоянного объекта схемы и не восстанавливается после сбоев, поэтому предназначен в первую очередь для отладки и проверки данных в топике.

Подробнее см. в разделе [{#T}](../../yql/reference/syntax/select/streaming.md).
