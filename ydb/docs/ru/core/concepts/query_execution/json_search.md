# Поиск по JSON-документам

Поиск по JSON-документам — это способ находить строки таблицы по содержимому колонок типа `Json` или `JsonDocument`: по наличию пути внутри документа, по значению атрибута, или по комбинации значений атрибутов. В отличие от обычной фильтрации по равенству или диапазону значений колонок таблицы, поиск по JSON работает с произвольными путями [JsonPath](../../yql/reference/builtins/json.md#jsonpath) внутри документа. Это удобно, когда у объектов разнородный или расширяемый набор атрибутов.

Для доступа к данным JSON в {{ ydb-short-name }} предусмотрены функции [JSON_EXISTS](../../yql/reference/builtins/json.md) и [JSON_VALUE](../../yql/reference/builtins/json.md). Поиск по данным JSON осуществляется путём использования этих функций в условии отбора, указанном в операторе `WHERE`. Для ускорения поиска записей в таблице по содержимому JSON-полей могут быть задействованы специализированные JSON-индексы.

Ключевая идея [JSON-индекса](../../dev/json-indexes.md) — построение [инвертированного индекса](https://ru.wikipedia.org/wiki/Инвертированный_индекс) по путям и парам «путь + значение». Каждый путь JSON-документа (а для проверок равенства — и скалярное значение по этому пути) кодируется в токен. Для каждого токена в индексе хранится список значений первичного ключа соответствующих строк таблицы. Запросы к индексу сводятся к инвертированному поиску по этим токенам — по той же схеме, что и у [базового полнотекстового индекса](../../dev/fulltext-indexes.md#basic), но с собственным токенизатором JSON.

JSON-индекс позволяет выполнять запросы вида «найти строки, где в JSON-поле есть значение V по пути P» без полного сканирования таблицы. Индекс сужает множество обрабатываемых строк по токенам, после чего движок исполнения запросов повторно проверяет полный предикат (пост-фильтр). Это позволяет использовать индекс не только для равенства, но и для сравнений вида `!=`, `<`, `>=`, `BETWEEN`, `starts with`, `like_regex` — индекс отсекает строки по токену пути, а итоговую точность сравнения обеспечивает пост-фильтр. Результат поиска по индексу всегда совпадает с выполнением того же запроса без индекса.
