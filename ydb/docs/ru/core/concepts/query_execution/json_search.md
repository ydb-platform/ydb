# Поиск по JSON

## Понятие поиска по JSON

**Поиск по JSON** — это способ находить строки таблицы по тому, что хранится *внутри* JSON-документа в колонке типа `Json` или `JsonDocument`: по существованию пути в документе и по значению, расположенному в указанном пути. Путь задаётся выражением [JsonPath](../../yql/reference/builtins/json.md#jsonpath), а проверки выполняются уже существующими функциями [JSON_EXISTS](../../yql/reference/builtins/json.md#json_exists) и [JSON_VALUE](../../yql/reference/builtins/json.md#json_value). Типичные сценарии:

* поиск документов с определённым вложенным полем;
* поиск документов, у которых поле по заданному пути равно нужному значению;
* фильтрация полуструктурированных данных без заранее зафиксированной схемы.

Ключевая идея — **[инвертированный индекс](https://ru.wikipedia.org/wiki/Инвертированный_индекс)**: каждый путь в JSON-дереве (а для скалярных значений — путь вместе со значением) становится отдельной записью, для которой хранится список строк (первичных ключей), где такой путь встречается. Поэтому запросы вида «найти все документы, где есть путь `$.a.b`» или «найти все документы, где `$.a.b` равно `10`» можно выполнять без полного сканирования таблицы.

В {{ ydb-short-name }} поиск по JSON можно выполнять двумя основными способами:

* **без индекса** — сканированием таблицы и применением `JSON_EXISTS` / `JSON_VALUE` к каждой строке. Подход прост, но плохо масштабируется: объём работы растёт вместе с размером таблицы.
* **с JSON-индексом** — созданием [JSON-индекса](../../dev/json-indexes.md) по JSON-колонке. Этот подход предназначен для масштабируемого поиска.

## Поиск по JSON с JSON-индексом {#json-search-index}

JSON-индекс строит инвертированный индекс по JSON-колонке и позволяет:

* находить строки по **существованию пути** через [JSON_EXISTS](../../yql/reference/builtins/json.md#json_exists);
* находить строки по **значению в пути** через [JSON_VALUE](../../yql/reference/builtins/json.md#json_value) или фильтр-предикат внутри `JSON_EXISTS`.

В отличие от [полнотекстового индекса](fulltext_search.md), JSON-индекс не требует обязательного указания `VIEW`. Он поддерживает два режима использования:

* **Автоматический выбор.** Достаточно написать обычное условие `WHERE` с `JSON_EXISTS` / `JSON_VALUE` по индексированной JSON-колонке — [оптимизатор](../glossary.md#optimizer) сам распознает такой предикат и заменит его на чтение по индексу.
* **Явное указание индекса.** JSON-индекс, как и любой другой [вторичный индекс](../glossary.md#secondary-index), можно принудительно задействовать, указав его имя в секции `VIEW IndexName`.

Эти два режима по-разному ведут себя, если предикат не удаётся превратить в обращение к индексу: при автоматическом выборе запрос просто выполняется без ускорения индексом (результат остаётся корректным), а при явном `VIEW` возвращается ошибка. Подробнее см. [{#T}](../../yql/reference/syntax/select/json_index.md).
