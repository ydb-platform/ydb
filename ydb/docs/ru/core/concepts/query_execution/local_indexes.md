# Локальные индексы

## Отличие от глобальных индексов

[Локальный индекс](../glossary.md#local-index) — вспомогательная структура, которая хранится вместе с данными таблицы и применяется на этапе чтения из хранилища. Она не материализует отдельную индексную таблицу и не подключается в запросе через `VIEW` (в отличие от глобальных индексов).

В {{ ydb-short-name }} глобальные индексы — это отдельные структуры с собственным ключом или представлением данных:

* [Вторичный индекс](../glossary.md#secondary-index) — для доступа к строкам по значениям, отличным от первичного ключа.
* [Векторный индекс](../../dev/vector-indexes.md) — для поиска по векторному сходству.
* [Полнотекстовый индекс](../../dev/fulltext-indexes.md) — инвертированный индекс для текстового поиска.

Локальные индексы работают как фильтры чтения основной таблицы: [оптимизатор запросов](optimizer.md) и слой хранения используют их, чтобы пропускать нерелевантные фрагменты данных при сканировании.

Сейчас в {{ ydb-short-name }} реализованы локальные [Блум-индексы](#bloom-skip-indexes); в будущем планируются и другие виды локальных индексов.

## Блум-индексы {#bloom-skip-indexes}

Блум-индексы — частный случай [локального индекса](../glossary.md#local-index), построенный на основе [фильтра Блума](https://ru.wikipedia.org/wiki/Фильтр_Блума).

При чтении для каждого фрагмента данных индекс проверяет, может ли в нём встречаться искомое значение (или набор n-грамм). Если фильтр сообщает, что значение точно не встречается, фрагмент пропускается без чтения проиндексированных колонок. Если фильтр «пропускает» проверку, значение может присутствовать — в том числе из-за ложноположительного срабатывания, — и фрагмент нужно читать. Это уменьшает объём фактически прочитанных данных при селективных запросах.

### Типы блум-индексов

* `bloom_filter` — строит фильтр по точным значениям индексируемой колонки. Подходит для условий равенства (`=`), проверки вхождения в список (`IN`) и других сравнений на равенство для поддерживаемых типов.
* `bloom_ngram_filter` — строит фильтр по n-граммам строковой колонки (`String`, `Utf8`). При поиске по подстроке или шаблону (`LIKE`) запрос разбивается на n-граммы; если в индексе фрагмента нет хотя бы одной из требуемых n-грамм, в нём не может быть искомой подстроки, и фрагмент пропускается. Поддерживается только в [колоночных таблицах](../glossary.md#column-oriented-table).

Подробнее о параметрах: [Блум-индексы](../../dev/bloom-skip-indexes.md), [CREATE TABLE](../../yql/reference/syntax/create_table/bloom_skip_index.md).

### Когда применять

* `bloom_filter` — когда в запросах часто фильтруют по конкретным значениям индексированной колонки (равенство, небольшой набор значений).
* `bloom_ngram_filter` — для строковых колонок, когда важны условия по подстроке или шаблону (`LIKE`), а не только точное равенство.
