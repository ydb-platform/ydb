# Просмотр информации о стриминговых запросах

Для просмотра информации о запросах используется системная таблица `streaming_queries`:

```sql
SELECT * FROM `.sys/streaming_queries`;
```

Схема таблицы:

- `query_name String` [Primary Key] - имя запроса
- `run Bool` - запущен ли запрос пользователем в данный момент
- `status String` - статус выполнения запроса, значение одно из:
  - `CREATING` - запрос создается
  - `CREATED` - запрос создан, но не запущен
  - `STARTING` - запрос запускается
  - `RUNNING` - запрос запущен
  - `STOPPING` - запрос останавливается
  - `STOPPED` - запрос остановлен
  - `COMPLETED` - запрос был завершён (например в случае запроса чтения из топика с лимитом)
  - `SUSPENDED` - запрос завершился с ошибкой и ожидает backoff для ретрая
  - `FAILED` - превышен лимит на число ретраев запроса, требуется ручной перезапуск
- `created_by String` - пользователь, который создал запрос
- `modified_by String` - пользователь который в последний раз менял запрос
- `started_by String` - пользователь который запустил запрос
- `stopped_by String` - пользователь который приостановил запрос
- `created_at Timestamp` - дата и время создания
- `modified_at Timestamp` - дата и время последней модификации
- `submitted_at Timestamp` - время постановки запроса на исполнение
- `started_at Timestamp` - время начала исполнения запроса
- `finished_at Timestamp` - время завершения запроса
- `last_execution_id String` - идентификатор последней операции исполнения запроса
- `has_saved_checkpoints Bool` - есть ли у запроса сохраненные чекпоинты
- `text String` - текст запроса
- `plan Json` - план запроса в формате json (для последнего execution)
- `ast String` - ast запроса (для последнего execution)
- `issues Json` - ошибки выполнения запроса в формате json (для последнего execution)
