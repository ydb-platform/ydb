# Система управления кластером (CMS)

Cluster Management System (CMS) — это [системная таблетка](../concepts/glossary.md#tablet-types), основная задача которой — гарантировать, что плановые работы по обслуживанию кластера (обновление, замена оборудования) не приведут к потере доступности данных из-за нарушения отказоустойчивости системы.

Вместо ручного анализа состояния сотен компонентов администратор создаёт в CMS задачу обслуживания на вывод узлов или дисков из строя. CMS проверяет, не нарушит ли выполнение этой задачи модель отказа групп хранения, State Storage или ресурсные лимиты. Если проверка проходит, CMS берёт на целевых компонентах эксклюзивные блокировки, временно исключая их из расчёта отказоустойчивости, что делает проведение работ безопасным.

Таким образом, CMS формализует и автоматизирует сложную логику принятия решений в распределённой системе, позволяя сочетать непрерывную доступность с необходимостью планового технического обслуживания.

## Динамическое управление конфигурацией

{{ ydb-short-name }} позволяет менять некоторые настройки напрямую через интерфейс администратора. Эти настройки не сохраняются при перезапуске кластера, для их постоянного применения следует изменять [динамическую конфигурацию](../devops/configuration-management/configuration-v1/dynamic-config.md).
