# Расположение информации в плане запроса

Графический SVG-план запроса с метриками получают после фактического выполнения запроса — см. [Получение графического плана при выполнении запроса через CLI](plans.md#svg-cli) и [в UI](plans.md#svg-ui). Ниже разобраны колонки диаграммы.

Во встроенных в документацию изображениях нет интерактивных элементов — откройте план в отдельной вкладке браузера, чтобы пользоваться сворачиванием стадий и всплывающими подсказками.

## Отдельные регионы (колонки) {#regions}

Ниже показан план простого запроса:

```sql
SELECT count(*) FROM lineitem
```

![План выполнения запроса](../../_assets/rts-count-lineitem.svg){inline=false}

План выводится в виде таблицы: слева — [структура стадий](structure.md#stages), справа — колонки с метриками и графиками.

| Колонка | Назначение | Подробнее |
| --- | --- | --- |
| `Query - ...` | Структура стадий и [операторов](../../concepts/glossary.md#operator) | [Структура плана](#structure) |
| `Rows` | Статистика по [операторам](../../concepts/glossary.md#operator) стадии | [Операторная статистика](#operators) |
| `Tasks` | Число [задач](../../concepts/glossary.md#task) и прогресс выполнения | [Число задач стадии и прогресс их выполнения](#progress) |
| `Statistics` | Суммарные метрики по [стадии](../../concepts/glossary.md#processing-stage) | [Метрики по стадиям](#statistics), [Визуализация метрик запроса](metrics.md) |
| Временная шкала | Графики метрик во времени | [Временные графики](#timeline), [Визуализация метрик запроса](metrics.md) |

## Структура плана {#structure}

Самая левая колонка — [структура стадий плана](structure.md). Каждая строка таблицы соответствует одной [стадии](../../concepts/glossary.md#processing-stage); [операторы](../../concepts/glossary.md#operator) внутри стадии — шаги обработки данных (подробнее — в [Структуре фактического плана запроса](structure.md)).

Заголовок начинается с общего префикса `Query`, вторая часть заголовка зависит от типа выполняемого запроса:

- `ResultSet` (как в примере выше) — запросы с выборкой (`SELECT`);
- `Sink` — модифицирующие запросы (`INSERT`, `UPDATE`, `DELETE`);
- `Precompute` — отдельно выполняемые фрагменты плана, см. [Составные структуры](structure.md#complex).

## Операторная статистика {#operators}

[Стадия](../../concepts/glossary.md#processing-stage) состоит из одного или нескольких [операторов](../../concepts/glossary.md#operator). Если хотя бы один [физический оператор](../../concepts/glossary.md#physical-operator) отдаёт статистику, колонка `Operators` заполнена; у стадий без отчётности по [операторам](../../concepts/glossary.md#operator) ячейка пуста. Содержимое колонки дополняет картину по [структуре стадии](structure.md) и агрегированным метрикам в `Stages`.

## Число задач стадии и прогресс их выполнения {#progress}

В {{ ydb-short-name }} запрос выполняется распределённо: одна и та же [стадия](../../concepts/glossary.md#processing-stage) может параллельно обрабатываться несколькими [задачами](../../concepts/glossary.md#task) на одном или нескольких [узлах](../../concepts/glossary.md#node) [кластера](../../concepts/glossary.md#cluster).

В колонке `Tasks` указано число [задач](../../concepts/glossary.md#task), запланированных для стадии. При наведении курсора во всплывающей подсказке видно, сколько из них уже завершилось. Слева в той же колонке прогресс показан пунктирной полосой.

Цвет фона колонки отражает интенсивность потребления CPU [стадией](../../concepts/glossary.md#processing-stage); см. [Потребление CPU](metrics.md#cpu).

## Метрики по стадиям {#statistics}

Колонка `Statistics` — агрегированные метрики по всем [задачам](../../concepts/glossary.md#task) стадии. Часть метрик описана в [Визуализация метрик запроса](metrics.md). Общее свойство: это рассчитанные значения в некоторый момент времени (как правило, в конце выполнения запроса).

Одинаковые метрики по разным стадиям приводятся к одному масштабу, чтобы их можно было быстро сравнивать между собой, например, потребление CPU или памяти. Подробнее — [Масштаб метрик](metrics.md#scale).

## Временные графики {#timeline}

Колонка с условным названием `Timeline` (это название не выводится в плане, на этом месте отображаются отметки времени) показывает те же самые величины, что и `Statistics`, но на всём интервале выполнения запроса. Шаг временной сетки подбирается автоматически. Длительность запроса указана в сером прямоугольнике в правом верхнем углу колонки.

## Интерактивные возможности {#interactive}

Откройте план запроса в отдельной вкладке браузера. У крупного запроса план занимает много строк и его удобно упрощать на экране:

- У каждой [стадии](../../concepts/glossary.md#processing-stage) кнопка ![](../../_assets/rts-button-minus.svg) сворачивает поддерево, начиная с этой стадии (кнопка меняется на ![](../../_assets/rts-button-plus.svg) для разворота).
- Кнопка ![](../../_assets/rts-button-up.svg) переводит [стадию](../../concepts/glossary.md#processing-stage) в режим `slim`: по высоте остаётся одна метрика (`Output`). Обычный вид возвращается при любом действии на стадии.
- В левом верхнем углу плана кнопка ![](../../_assets/rts-button-up.svg) включает режим `slim` для всех [стадий](../../concepts/glossary.md#processing-stage), кнопка ![](../../_assets/rts-button-down.svg) возвращает полный вид и разворачивает все поддеревья.
