# Как получить графическое представление плана

## UI {#ui}

Чтобы в [{{ ydb-short-name }} UI](../../reference/embedded-ui/ydb-monitoring.md) отображался графический план выполнения запроса, включите настройку ```Experiments | Execution plan```.

![Включение графического представления в UI](../../_assets/rts-ui-settings.png)

После этого выполните запрос, например:

```sql
SELECT count(*) FROM lineitem
```

В дополнительном меню справа (кнопка [...]) появятся пункты:

- ```Open Execution Plan``` — открыть план в новой вкладке браузера;
- ```Download Execution Plan``` — сохранить план в файл.

Там же есть пункт ```Download Diagnostics```.

![Просмотр и сохранение плана](../../_assets/rts-get-execution-plan.png)

Для этого запроса план будет выглядеть примерно так (конкретные значения метрик зависят от конфигурации и нагрузки кластера):

![План выполнения запроса](../../_assets/rts-count-lineitem.svg){inline=false}

Об элементах диаграммы и их назначениях подробнее написано в статье [Расположение информации на плане](layout.md). Чтобы разобраться в структуре плана и прохождении данных по стадиям, перейдите к разделу [Структура плана запроса](structure.md). Ниже описано получение плана из командной строки, если UI недоступен.

## ydb cli {#cli}

Способы сохранения графического плана файлом в {{ ydb-short-name }} CLI:

- Для сценариев вроде [TPC-H](../../reference/ydb-cli/workload-tpch.md) используйте [команду workload](../../reference/ydb-cli/commands/workload/index.md) с опцией ```--plan```: ```ydb workload tpch run --plan filename```. На диске появятся файлы с базовым именем ```filename``` в нескольких форматах; расширение ```.svg``` соответствует графическому представлению.

- Для произвольных запросов: ```ydb workload query run --plan filename```.
