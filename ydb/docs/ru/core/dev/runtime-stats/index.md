# Статистика выполнения запросов и её анализ

При выполнении пользовательского запроса {{ ydb-short-name }} фиксирует статистику его исполнения. Она помогает исследовать план выполнения запроса, находить и устранять узкие места. Например, выбирая более оптимальные решения. В результате можно существенно сократить время работы запроса. Особенно это проявляется в [аналитических (OLAP) запросах](../../concepts/datamodel/_includes/table.md#column-oriented-tables).

Статистика работы запроса доступна вместе с планом его выполнения в [формате JSON](https://ru.wikipedia.org/wiki/JSON). Большое количество метрик затрудняет его анализ и понимание, поэтому была разработана визуализация в [графическом формате SVG](https://ru.wikipedia.org/wiki/SVG). В данном разделе содержится информация об этом визуальном формате.

В качестве примера в данном разделе используется набор данных от [промышленного бенчмарка TPC-H scale 100](../../reference/ydb-cli/workload-tpch.md). Описание структуры таблиц и связей между ними вы можете найти в спецификации данного стандарта. Для иллюстрации мы использовали некоторые адаптированные из бенчмарка запросы, а также оригинальные, текст которых был специально подготовлен для иллюстрации материала. Текст каждого запроса приведён рядом с его планом выполнения.

Данный раздел рассчитан, главным образом, на анализ сложных аналитических запросов. Поэтому материал будет подразумевать [колоночный формат таблиц хранения данных](../../concepts/datamodel/_includes/table.md#column-oriented-tables).

Формат графического представления спроектирован таким образом, чтобы акцентировать внимание на наиболее нужных и полезных элементах. При этом опытный разработчик сможет получить из него больше информации, обращая внимание на дополнительные элементы и их свойства.
