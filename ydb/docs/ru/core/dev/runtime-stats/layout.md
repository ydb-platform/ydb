# Расположение информации на плане

## Отдельные регионы (колонки) {#regions}

Ниже показан план простого запроса:

```sql
SELECT count(*) FROM lineitem
```

![План выполнения запроса](../../_assets/rts-count-lineitem.svg){inline=false}

План выводится в виде таблицы: слева — древовидная структура стадий, справа — несколько колонок с метриками и графиками. Колонки с данными:

- `ResultSet` — структура запроса;
- `Operators` — операторная статистика;
- `Tasks` — число задач стадии и индикатор их завершения;
- `Stages` — агрегированные метрики по стадиям;
- `Timeline` — те же метрики во времени.

## Структура запроса {#structure}

Самая левая колонка — структура плана. Подробнее о ней написано в разделе [Структура плана запроса](structure.md). Здесь достаточно того, что план — дерево стадий, а каждая строка таблицы соответствует одной стадии.

Заголовок колонки зависит от типа запроса:

- `ResultSet` (как в примере выше) — запросы с выборкой (`SELECT`);
- `Sink` — модифицирующие запросы (`INSERT`, `UPDATE`, `DELETE`);
- `Precompute` — отдельно выполняемые фрагменты плана, см. [Прекомпьюты](structure.md#precomputes).

## Операторная статистика {#operators}

Стадия состоит из одного или нескольких операторов. Если оператор отдаёт статистику, она показывается в колонке `Operators`.

## Прогресс выполнения задач {#progress}

В {{ ydb-short-name }} запрос выполняется распределённо: одна и та же стадия может параллельно обрабатываться несколькими задачами (tasks) на одном или нескольких узлах кластера.

В колонке `Tasks` указано число задач, запланированных для стадии. При наведении курсора во всплывающей подсказке видно, сколько из них уже завершилось. Слева в той же колонке прогресс показан пунктирной полосой.

Цвет фона колонки отражает интенсивность потребления CPU стадией; см. [Потребление CPU](metrics.md#cpu).

## Детальная информация по стадиям {#stages}

Колонка `Stages` — агрегированные метрики по всем задачам стадии. Часть метрик описана в [Визуализация метрик](metrics.md). Общее свойство: это снимок итоговых значений на момент времени (как правило, к концу выполнения запроса).

Одинаковые метрики по разным стадиям приводятся к одному масштабу, чтобы быстро сравнивать, например, расход CPU или памяти. Подробнее — [Масштаб метрик](metrics.md#scale).

## Временные графики {#timeline}

Колонка `Timeline` показывает те же величины, что и `Stages`, но на всём интервале выполнения запроса. Шаг временной сетки подбирается автоматически. Длительность запроса указана в сером прямоугольнике в левом верхнем углу колонки.

## Интерактивные возможности {#interactive}

{% note info %}

Интерактивные элементы на встроенных в документацию изображениях недоступны. Откройте визуализацию плана в отдельной вкладке браузера.

{% endnote %}

У крупного запроса план занимает много строк и его удобно упрощать на экране:

- У каждой стадии кнопка ![](../../_assets/rts-button-minus.svg) сворачивает поддерево, начиная с этой стадии (кнопка меняется на ![](../../_assets/rts-button-plus.svg) для разворота).
- Кнопка ![](../../_assets/rts-button-up.svg) переводит стадию в режим `slim`: по высоте остаётся одна метрика (output). Обычный вид возвращается при любом действии на стадии.
- В левом верхнем углу плана кнопки ![](../../_assets/rts-button-up.svg) включают режим `slim` для всех стадий, ![](../../_assets/rts-button-down.svg) возвращают полный вид и разворачивают все поддеревья.
