# Обзор

При выполнении запроса {{ ydb-short-name }} собирает **статистику исполнения**: длительность шагов, объёмы переданных данных, ожидания и т.п. В других разделах документации слово «статистика» часто относится к **статистике источников** (таблиц, колонок) — тем данным, на которые опирается оптимизатор при выборе плана. Здесь и далее речь только о **статистике времени выполнения** — измерениях по одному конкретному прогону запроса.

По ним удобно разбирать план, искать узкие места и проверять эффект правок текста запроса. Наибольший выигрыш по времени обычно даёт такая работа в [аналитических (OLAP) сценариях](../../concepts/datamodel/table.md).

Статистика исполнения поставляется вместе с планом в [JSON](https://ru.wikipedia.org/wiki/JSON). Сырых метрик много, разбор вручную утомителен, поэтому для плана есть визуализация в [SVG](https://ru.wikipedia.org/wiki/SVG); эта глава описывает именно её.

В примерах используется набор данных [TPC-H, scale factor 100](../../reference/ydb-cli/workload-tpch.md). Схему таблиц и связи см. в спецификации TPC-H. Часть запросов взята из бенчмарка с адаптацией под {{ ydb-short-name }}, часть приведена отдельно для иллюстрации приёмов чтения плана; SQL всегда дан рядом со схемой выполнения.

График плана рассчитан на быстрый обзор ключевых метрик; дополнительные элементы и подсказки дают больше деталей тем, кто уже ориентируется в формате.
