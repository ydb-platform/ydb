# Автоматическое партицирование по нагрузке (load-based) {#auto-by-load}

Режим **`AUTO_PARTITIONING_BY_LOAD`** позволяет разделять партицию, когда она **перегружена по CPU**, даже если размер партиции ещё не большой. По умолчанию для строковых таблиц этот режим **выключен** (`DISABLED`); для высокой конкурентной записи/чтения по узкому диапазону ключей его обычно имеет смысл **включить** вместе с осмысленным [`AUTO_PARTITIONING_MIN_PARTITIONS_COUNT`](../../../../concepts/datamodel/table.md#auto_partitioning_min_partitions_count).

Каноническое описание порогов и алгоритма (сэмпл ключей, учёт реплик, merge при низкой загрузке) — в [{#T}](../../../../concepts/datamodel/table.md#auto_partitioning_by_load).

### Типичные симптомы перегруженной партиции

* высокая загрузка CPU на отдельных партициях при умеренной общей загрузке кластера;
* рост задержек на «горячих» ключах;
* ошибки вида **`STATUS_OVERLOADED`** при записи в перегруженную партицию.

Один data shard использует **не более одного ядра CPU** для операций изменения данных на партиции: простое добавление CPU на узел не снимает bottleneck с одной партиции — нужны split и/или более удачный ключ.

Дополнительные шаги диагностики перегруженного shard — в [{#T}](../../../../troubleshooting/performance/schemas/overloaded-shards.md).
