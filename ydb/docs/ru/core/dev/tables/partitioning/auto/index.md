# Автоматическое партицирование {#automatic-partitioning}

Для строковых таблиц {{ ydb-short-name }} может автоматически **разделять** (split) и **объединять** (merge) партиции, чтобы подстроиться под объём данных и интенсивность запросов. Это задаётся параметрами `AUTO_PARTITIONING_*` в описании таблицы (см. [{#T}](../../../../concepts/datamodel/table.md#partitioning_row_table)).

Материалы в этом подразделе:

* [{#T}](by-size.md) — режим по объёму (`AUTO_PARTITIONING_BY_SIZE`, порог размера партиции);
* [{#T}](by-load.md) — режим по нагрузке (`AUTO_PARTITIONING_BY_LOAD`, пороги CPU);
* [{#T}](limits.md) — внутренние ограничения, лимиты min/max и типичные конфликты со split/merge.

Типичное время операции разделения или слияния — порядка **500 мс**; в этот интервал данные по затронутой партиции могут быть кратковременно недоступны для чтения и записи (подробности — в [{#T}](../../../../concepts/datamodel/table.md#partitioning_row_table)).
