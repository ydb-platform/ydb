# Как работает партицирование {#how-partitioning-works}

Строковая таблица в {{ ydb-short-name }} может быть разбита на **партиции** — непересекающиеся диапазоны значений [первичного ключа](../../../concepts/datamodel/table.md#primary-key). Каждая партиция обслуживается отдельной таблеткой [data shard](../../../concepts/glossary.md#data-shard); партиции могут находиться на разных узлах кластера и переноситься между ними при балансировке и отказах.

Подробное описание модели, параметров `WITH (...)` и поведения при росте данных и нагрузки приведено в статье [{#T}](../../../concepts/datamodel/table.md) — разделы [Партиционирование строковой таблицы](../../../concepts/datamodel/table.md#partitioning_row_table) и перечень параметров [`AUTO_PARTITIONING_*`](../../../concepts/datamodel/table.md#partitioning_row_table).

Дальше в этом подразделе:

* [{#T}](auto/index.md) — режимы **по объёму** и **по нагрузке**, а также ограничения автоматического партицирования;
* [{#T}](choosing-partition-count.md) — выбор минимального и максимального числа партиций;
* [{#T}](anti-patterns/index.md) — типичные ошибки схемы и конфигурации.

{% note info %}

Колоночные таблицы партицируются иначе и **не поддерживают** автоматического репартицирования по тем же правилам, что строковые. См. [{#T}](../../primary-key/column-oriented.md).

{% endnote %}
