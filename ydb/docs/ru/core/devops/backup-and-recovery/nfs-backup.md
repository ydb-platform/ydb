# Резервное копирование через NFS

В этом разделе описан практический сценарий резервного копирования и восстановления данных {{ ydb-short-name }} с использованием сетевой файловой системы (NFS). В отличие от [dump/restore](../../reference/ydb-cli/export-import/tools-dump.md), команды `export nfs` / `import nfs` выполняются на стороне сервера, создают согласованный снимок данных и поддерживают шифрование и сжатие.

## Предварительные требования {#prerequisites}

### NFS

NFS-директория должна быть смонтирована на **всех узлах** кластера {{ ydb-short-name }} по одному и тому же абсолютному пути. Процесс {{ ydb-short-name }} должен иметь права на запись в эту директорию.


### {{ ydb-short-name }} CLI

Установите [{{ ydb-short-name }} CLI](../../reference/ydb-cli/index.md) и настройте [профиль соединения](../../reference/ydb-cli/profile/index.md).

## Экспорт данных {#export}

### Полная выгрузка базы данных

Для выгрузки всей базы данных выполните:

```bash
{{ ydb-cli }} -p quickstart export nfs \
  --fs-path /mnt/ydb_backups/full_backup_2025_01_15
```

Команда ставит операцию экспорта в очередь и возвращает идентификатор операции. Фактическая выгрузка выполняется сервером асинхронно.

## Мониторинг операций {#monitoring}

### Отслеживание прогресса

Запомните идентификатор операции из вывода команды `export nfs` и используйте его для проверки статуса:

```bash
{{ ydb-cli }} -p quickstart operation get "ydb://export/6?id=281474976788395&kind=fs"
```

Операция завершена, когда поле `progress` имеет значение `Done` (в режиме `pretty`) или `PROGRESS_DONE` (в режиме `proto-json-base64`).

### Список всех операций экспорта

```bash
{{ ydb-cli }} -p quickstart operation list export/nfs
```

### Очистка завершённых операций

После успешного экспорта удалите операцию из списка:

```bash
{{ ydb-cli }} -p quickstart operation forget "ydb://export/6?id=281474976788395&kind=fs"
```

## Восстановление из резервной копии {#restore}

Для загрузки данных из резервной копии на NFS используйте команду `import nfs`:

```bash
{{ ydb-cli }} -p quickstart import nfs \
  --fs-path /mnt/ydb_backups/full_backup_2025_01_15
```

{% note warning %}

Команда `import nfs` создаёт объекты заново. Для успешного выполнения ни один из импортируемых объектов не должен существовать в базе данных.

{% endnote %}
