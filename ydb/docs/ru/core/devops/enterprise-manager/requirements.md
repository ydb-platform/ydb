# Предварительные требования {#requirements}

Перед установкой {{ ydb-short-name }} Enterprise Manager убедитесь, что выполнены все перечисленные ниже требования.

## База данных {{ ydb-short-name }} {#ydb-database}

{{ ydb-short-name }} Enterprise Manager использует базу данных {{ ydb-short-name }} для хранения метаданных двух основных компонентов — Gateway и Control Plane. Для установки вам потребуется следующая информация о базе данных:

* **Endpoint** — адрес подключения к базе данных (например, `grpcs://ydb-node01.ru-central1.internal:2135`).
* **Имя пользователя и пароль** — учётные данные для получения токена аутентификации.

## Конфигурация узлов {{ ydb-short-name }} {#node-configuration}

Убедитесь, что узлы {{ ydb-short-name }}, предназначенные для работы с динамическими слотами, настроены правильно:

* Время синхронизировано между всеми узлами кластера.
* Имена хостов соответствуют их FQDN.
* TLS-сертификаты установлены на каждом узле.
* Бинарные файлы {{ ydb-short-name }} расположены в директории `/opt/ydb`.
* На кластере {{ ydb-short-name }} настроена [динамическая конфигурация](../configuration-management/index.md).

## Программное обеспечение на управляющей машине {#control-machine}

На машине, с которой будет выполняться установка, необходимы:

* [Ansible](https://www.ansible.com/) (рекомендуемые версии — от 2.11 до 2.18).
* SSH-доступ ко всем серверам кластера.
