# Совместимость {{ ydb-short-name }} с PostgreSQL

{% include [./_includes/alert.md](./_includes/alert_preview.md) %}

Совместимость с PostgreSQL — это механизм выполнения SQL-запросов в диалекте PostgreSQL на инфраструктуре {{ ydb-short-name }} с использованием стандартных инструментов и API {{ ydb-short-name }}. Благодаря этой возможности можно разрабатывать запросы на привычном синтаксисе PostgreSQL и получать такие преимущества {{ ydb-short-name }}, как горизонтальная масштабируемость и отказоустойчивость.

Совместимость с PostgreSQL упрощает миграцию приложений, ранее работавших в экосистеме PostgreSQL. Сейчас поддерживается ограниченное количество инструкций и функций PostgreSQL 16. Совместимость позволяет переключаться с PostgreSQL на {{ ydb-short-name }} без изменения кода проекта (в случае полной поддержки совместимостью используемых в проекте SQL-конструкций), изменив способ отправки запросов в {{ ydb-short-name }}.

Работа совместимости с PostgreSQL может быть описана следующим образом:

1. Приложение отправляет запросы в диалекте PostgreSQL в {{ ydb-short-name }} через стандартные инструменты {{ ydb-short-name }}.
2. Процессор запросов переводит запросы PostgreSQL в YQL AST.
3. После обработки запросов результаты компилируются и возвращаются приложению. Во время обработки запрос может быть распараллелен и выполнен на любом количестве узлов {{ ydb-short-name }}.
