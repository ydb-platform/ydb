# Bloom skip индекс — настройка параметров

Этот рецепт показывает базовый подход к настройке параметров локальных Bloom skip индексов после первичного запуска.

## Когда настраивать

Если запросы читают слишком много данных, попробуйте уменьшить `false_positive_probability`.  
Если индекс занимает слишком много места, попробуйте увеличить `false_positive_probability`.

## Пример изменения параметров `bloom_ngram_filter`

```yql
ALTER TABLE `/Root/events` ALTER INDEX idx_msg SET (
    ngram_size = 4,
    false_positive_probability = 0.005,
    case_sensitive = false
);
```

## Практические рекомендации

* Начните с `false_positive_probability = 0.01`, затем корректируйте по фактическим метрикам чтения и размеру индекса.
* `ngram_size` обычно начинают с `3`; увеличение значения может сделать фильтрацию более строгой для более длинных подстрок.
* Меняйте параметры по одному шагу и сравнивайте результат на одной и той же нагрузке.
