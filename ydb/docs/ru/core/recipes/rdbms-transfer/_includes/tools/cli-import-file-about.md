## CLI import file {#about}

**YDB CLI import file** — команда `ydb import file` для пакетной загрузки данных из локальных файлов в **уже существующую** таблицу {{ ydb-short-name }}.

### Когда использовать

* Нужен простой офлайн-пайплайн: выгрузка из СУБД → файл → {{ ydb-short-name }}.
* Данные можно экспортировать в CSV, TSV, JSON или Parquet.
* Не требуется прямое JDBC-подключение к источнику с хоста {{ ydb-short-name }}.

### Системные требования

| Компонент | Требование |
| --- | --- |
| ОС | Linux, macOS или Windows (WSL); 64-bit |
| {{ ydb-short-name }} CLI | Установлен, в `PATH` |
| Диск | Место под промежуточный файл экспорта |
| Память | CLI читает файл порциями (`--batch-bytes`); весь файл не загружается в RAM |

### Поддерживаемые форматы

CSV, TSV, JSON, Parquet — см. [import file](../../../../reference/ydb-cli/export-import/import-file.md).

### Ограничения

* Целевая таблица и схема должны быть созданы заранее (или через отдельный DDL-пайплайн).
* Импорт выполняется через `BulkUpsert` — строки с совпадающим PK перезаписываются.
