# cms_config

[CMS](../../concepts/glossary.md#cms) — компонент {{ ydb-short-name }}, с помощью которого можно выполнять безопасное обслуживание кластера, например, постепенный перезапуск узлов без воздействия на пользовательские нагрузки. Поведение CMS конфигурируется в секции `cms_config` [конфигурации](../../reference/configuration) {{ ydb-short-name }}.

## Приостановка обслуживания кластера

В экстренных ситуациях возможно временно приостановить обслуживание кластера {{ ydb-short-name }}. Это можно сделать на уровне системы, выставив в конфигурации следующий флаг:

```yaml
cms_config:
  disallow_maintenance: true
```

Любые новые попытки обслуживания будут отклонены. Уже начатое обслуживание останется без изменений.
