# feature_flags

Секция конфигурации `feature_flags` включает или отключает определённые функции {{ ydb-short-name }} с помощью булевых флагов. Эти флаги управляют экспериментальными функциями, оптимизациями производительности и настройками совместимости, которые можно переключать без изменения кода {{ ydb-short-name }}. Общую информацию о конфигурации {{ ydb-short-name }} см. в разделе [{#T}](../../devops/configuration-management/index.md).

{% note warning %}

Если функция отключена по умолчанию, у этого всегда есть причина. Это может быть новая функция, которая ещё не подтверждена как готовая для использования продакшене, экспериментальная функция, устаревшая функция или функция для специфических сценариев использования с ограничениями. Таким образом, эта секция конфигурации предназначена в основном для целей тестирования.

**Включайте функции {{ ydb-short-name }} вручную в production-средах только на свой страх и риск.**

{% endnote %}


## Доступные настройки

Следующие разделы содержат список всех доступных функциональных флагов, их значения по умолчанию, требования к перезапуску и подробные описания. Флаги, помеченные как «Устарел», поддерживаются для обратной совместимости и не должны использоваться в новых кластерах.


### enable_separate_solomon_shard_for_pdisk

**Описание:** Использует выделенный шард мониторинга для сбора метрик [PDisk](../../concepts/glossary.md#pdisk), улучшая производительность мониторинга и снижая влияние на операции хранения. Это изолирует метрики Solomon (системы мониторинга) от основной функциональности PDisk.

**По умолчанию:** `true`.

**Требует перезапуска:** нет.


### use_forseti_scheduler_by_default_for_pdisk

**Описание:** Включает планировщик ввода-вывода Forseti для операций PDisk, который обеспечивает лучшую справедливость и контроль задержки для дисковых операций. Forseti — это пользовательский дисковый планировщик {{ ydb-short-name }}, оптимизированный для рабочих нагрузок баз данных.

**По умолчанию:** `true`.

**Требует перезапуска:** нет.


### enable_separate_trim_thread_for_pdisk

**Описание:** Выполняет операции TRIM SSD в выделенном потоке, предотвращая блокировку других операций ввода-вывода PDisk командами TRIM. Это улучшает производительность на SSD, позволяя сборку мусора без влияния на задержку чтения/записи.

**По умолчанию:** `true`.

**Требует перезапуска:** нет.


### enable_separate_submit_thread_for_pdisk

**Описание:** Обрабатывает отправку запросов ввода-вывода в отдельном потоке, снижая задержку и улучшая пропускную способность, предотвращая блокировку основного цикла обработки PDisk операциями отправки.

**По умолчанию:** `true`.

**Требует перезапуска:** нет.


### enable_per_owner_chunk_limit_for_pdisk

**Описание:** Применяет индивидуальные ограничения выделения блоков для каждого владельца [VDisk](../../concepts/glossary.md#vdisk) на PDisk, предотвращая монополизацию дискового пространства любым отдельным владельцем. Это обеспечивает лучшую изоляцию ресурсов между различными компонентами базы данных.

**По умолчанию:** `false`.

**Требует перезапуска:** нет.


### trim_entire_device_on_startup

**Описание:** Выполняет полную операцию TRIM устройства во время запуска PDisk для обеспечения оптимальной производительности SSD. Предупреждение: это может значительно увеличить время запуска на больших устройствах.

**По умолчанию:** `false`.

**Требует перезапуска:** нет.


### enable_chunk_grace_for_pdisk

**Описание:** Позволяет временное превышение ограничений блоков при определённых условиях, обеспечивая гибкость во время сценариев высокой нагрузки при сохранении общих ограничений ресурсов.

Это предоставляет более строгие гарантии согласованности для операций DDL. Подробнее о моделях согласованности см. в разделе [{#T}](../../concepts/transactions.md).

**Требует перезапуска:** нет.

**Описание:** Включает поддержку шифрования в операциях экспорта, защищая экспортированные данные шифрованием во время передачи и хранения для улучшенной безопасности и соответствия.

**По умолчанию:** `false`.

**Требует перезапуска:** нет.


### enable_alter_database

**Описание:** Включает DDL-операции `ALTER DATABASE`, позволяя модификацию настроек и свойств на уровне базы данных времени выполнения без требования перезапуска кластера.

**По умолчанию:** `false`.

**Требует перезапуска:** нет.


### enable_export_auto_dropping

**Описание:** Автоматически очищает временные таблицы и директории, созданные во время операций экспорта в S3, снижая затраты на хранение путём удаления промежуточных артефактов экспорта.

**По умолчанию:** `false`.

**Требует перезапуска:** нет.


### enable_throttling_report

**Описание:** Включает подробные отчёты о троттлинге для мониторинга и анализа паттернов троттлинга запросов, помогая выявлять узкие места производительности и ограничения ресурсов.

**По умолчанию:** `true`.

**Требует перезапуска:** нет.


### enable_node_broker_delta_protocol

**Описание:** Включает дельта-протокол в NodeBroker для эффективных инкрементальных обновлений конфигурации узлов, снижая сетевые накладные расходы и улучшая производительность координации кластера.

**По умолчанию:** `false`.

**Требует перезапуска:** нет.


### enable_access_to_index_impl_tables

**Описание:** Позволяет использование ведомых (реплик для чтения) для покрытых запросов вторичных индексов, улучшая производительность чтения путём распределения нагрузки поиска индексов между несколькими репликами.

**По умолчанию:** `true`.

**Требует перезапуска:** нет.


### enable_add_unique_index

**Описание:** Включает создание уникальных вторичных индексов на существующих таблицах, предоставляя дополнительные ограничения уникальности и улучшая производительность запросов для поиска уникальных значений.

**По умолчанию:** `false`.

**Требует перезапуска:** нет.


### enable_shared_metadata_accessor_cache

**Описание:** Устарел.

**По умолчанию:** `true`.

**Требует перезапуска:** нет.


### require_db_prefix_in_secret_name

**Описание:** Требует префикс базы данных в именах секретов для лучшей изоляции пространства имён и безопасности, предотвращая случайный доступ к секретам из разных баз данных.

**По умолчанию:** `false`.

**Требует перезапуска:** нет.


### enable_system_names_protection

**Описание:** Защищает зарезервированные системой имена от использования пользовательскими объектами, предотвращая конфликты имён и обеспечивая целостность системы путём резервирования критически важных пространств имён.

**По умолчанию:** `false`.

**Требует перезапуска:** нет.


### enable_real_system_view_paths

**Описание:** Использует реальные пути файловой системы для системных представлений вместо виртуальных путей, предоставляя прямой доступ к системным метаданным через стандартные механизмы разрешения путей.

**По умолчанию:** `false`.

**Требует перезапуска:** да.


### enable_cs_schemas_collapsing

**Описание:** Включает оптимизацию схлопывания схем ColumnShard, которая снижает накладные расходы метаданных путём слияния совместимых версий схем, улучшая эффективность хранения и производительность запросов.

**По умолчанию:** `true`.

**Требует перезапуска:** нет.


### enable_move_column_table

**Описание:** Включает перемещение столбцовых таблиц между различными местоположениями хранения или конфигурациями без пересоздания данных, поддерживая гибкое управление жизненным циклом данных.

**По умолчанию:** `false`.

**Требует перезапуска:** нет.


### enable_arrow_result_set_format

**Описание:** Включает формат Apache Arrow для наборов результатов запросов, предоставляя эффективную передачу столбцовых данных и лучшую интеграцию с аналитическими инструментами и фреймворками.

**По умолчанию:** `false`.

**Требует перезапуска:** нет.


### enable_cs_overloads_subscription_retries

**Описание:** Включает механизмы повторных попыток для подписок на перегрузки ColumnShard, улучшая устойчивость при мониторинге условий перегрузки столбцового хранилища путём автоматических повторных попыток неудачных попыток подписки.

**По умолчанию:** `false`.

**Требует перезапуска:** нет.


### enable_topic_compactification_by_key

**Описание:** Включает уплотнение топиков на основе ключей сообщений, сохраняя только последнее сообщение для каждого ключа для снижения использования хранилища при сохранении самого последнего состояния для каждого ключа.

**По умолчанию:** `false`.

**Требует перезапуска:** нет.


## Пример конфигурации

Для включения определённых функций в вашем развёртывании {{ ydb-short-name }} добавьте желаемые функциональные флаги в файл конфигурации. Следующий пример показывает, как включить поддержку векторных индексов и автоматическое партиционирование топиков для Change Data Capture (CDC):

```yaml
feature_flags:
  enable_vector_index: true
  enable_topic_autopartitioning_for_cdc: true
```
