# node_broker_config

Секция `node_broker_config` настраивает работу [Node Broker](../../concepts/glossary.md#node-broker) в кластере {{ ydb-short-name }}. В этой секции настраиваются длительность эпох и аренды [динамических узлов](../../concepts/glossary.md#dynamic), а также стабильные имена динамических узлов.

## Параметры

| Параметр | Значение по умолчанию | Описание |
|---|---|---|
| `epoch_duration` | `3600000000` (1 час) | Длительность эпохи Node Broker в микросекундах. На границе эпохи Node Broker пересчитывает состояние зарегистрированных динамических узлов и продлевает их аренду. Минимальное значение — 5 минут (`300000000` µs). |
| `lease_duration` | `259200000000` (72 часа) | Дополнительный срок в микросекундах, на который продлевается аренда динамического узла сверх конца следующей эпохи. Учитывается только при включённом функциональном флаге [enable_node_broker_long_lease](feature_flags.md). Минимальное значение — 5 минут (`300000000` µs). |
| `stable_node_name_prefix` | `slot-` | Префикс [стабильных имён динамических узлов](#stable-node-names). |

## Пример

```yaml
node_broker_config:
  epoch_duration: 3600000000
  lease_duration: 259200000000
  stable_node_name_prefix: "slot-"
```

## Стабильные имена узлов {#stable-node-names}

Node Broker присваивает имена динамическим узлам при их регистрации в кластере. По умолчанию имя динамического узла состоит из имени хоста и номера порта, на котором работает динамический узел.

В динамической среде, где имена хостов часто меняются, например в Kubernetes, использование имени хоста и порта приводит к неконтролируемому росту количества уникальных имен динамических узлов, даже для базы данных с небольшим количеством динамических узлов. Подобное поведение может быть нежелательным для системы time series мониторинга — количество метрик растет некотролируемо. Для решения этой проблемы администратор системы может настроить *стабильные* имена динамических узлов.

Стабильное имя идентифицирует динамический узел в рамках тенанта. Оно состоит из префикса и порядкового номера динамического узла в рамках тенанта. Если динамический узел был выключен, то по истечении таймаута его стабильное имя может быть занято новым динамическим узлом, обслуживающим того же тенанта.

Чтобы включить стабильные имена динамических узлов, необходимо добавить в конфигурацию кластера следующее:

```yaml
feature_flags:
  enable_stable_node_names: true
```

По умолчанию, префиксом является `slot-`. Для того, чтобы переопределить префикс, необходимо добавить в конфигурацию кластера следующее:

```yaml
node_broker_config:
  stable_node_name_prefix: <новый префикс>
```
