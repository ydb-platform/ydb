Если в конфигурации кластера в секции `client_certificate_authorization` включён параметр [`request_client_certificate`](../../configuration/client_certificate_authorization.md), сервер также запрашивает клиентский сертификат при TLS-handshake. В этом случае в качестве эндпойнта необходимо указывать gRPCs.

- `--client-cert-file <filename>` : файл клиентского сертификата (PEM или PKCS#12)

- `--ca-file <filename>` : файл корневого PEM-сертификата для проверки серверного сертификата

- `--client-cert-key-file <filename>` : файл закрытого ключа клиентского сертификата; указывается, если сертификат и ключ переданы отдельными файлами

- `--client-cert-key-password-file <filename>` : файл с паролем к закрытому ключу, если ключ зашифрован; если ключ зашифрован, а опция не указана, пароль будет запрошен интерактивно

Если другие опции [аутентификации](#authentication) не заданы, сервер использует переданный клиентский сертификат для [аутентификации по клиентскому сертификату](../../../../security/authentication.md#client-certificate). Если одновременно указаны `--client-cert-file` и другой режим аутентификации, сертификат по-прежнему передаётся при TLS-handshake, но для аутентификации запросов используется [аутентификационный токен](../../../../concepts/glossary.md#auth-token), а не сертификат.
