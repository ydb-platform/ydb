# IAM-аутентификация с делегированием сервисному аккаунту

IAM-аутентификация с делегированием сервисному аккаунту представляет собой схему, при которой {{ ydb-full-name }} автоматически получает свежие IAM-токены от имени [сервисного аккаунта](https://yandex.cloud/ru/docs/iam/concepts/users/service-accounts) на стороне базы. Её имеет смысл использовать для длительных или постоянно работающих операций, таких как потоковый запрос, асинхронная репликация или трансфер данных между базами. Стандартная аутентификация по [IAM-токену](https://yandex.cloud/ru/docs/iam/concepts/authorization/iam-token) для них неудобна, поскольку облачные токены имеют ограниченный срок действия, а передавать свежий токен с каждой операцией такого ресурса невозможно.

Эта схема используется в:

- [Асинхронной репликации](../concepts/async-replication.md) и [трансфере данных](../concepts/transfer.md) между базами.
- [Внешних источниках данных](../concepts/datamodel/external_data_source.md) для долгосрочных запросов.

## Как работает {#how-it-works}

Делегирование работает в два шага. При создании ресурса однократно используется IAM-токен сервисного аккаунта (`INITIAL_TOKEN_SECRET_PATH`): он подтверждает доступ к целевой базе и позволяет определить её идентификатор в облаке. После этого начальный токен больше не нужен и может истечь, дальнейшую аутентификацию {{ ydb-full-name }} выполняет сам, автоматически получая свежие IAM-токены от имени сервисного аккаунта (`SERVICE_ACCOUNT_ID`) на всё время существования ресурса.

## Подготовка сервисного аккаунта {#service-account}

В облаке необходимо подготовить сервисный аккаунт, которому будут делегированы права доступа к целевой базе:

1. Назначьте сервисному аккаунту нужные роли на целевой базе данных. Конкретный набор ролей определяется сценарием использования и описан в документации соответствующей фичи.
2. Включите [делегирование сервисного аккаунта](https://yandex.cloud/ru/docs/iam/concepts/service-control) для целевой базы данных.

## Параметры {#parameters}

- `SERVICE_ACCOUNT_ID`: идентификатор подготовленного сервисного аккаунта.
- `INITIAL_TOKEN_SECRET_PATH`: имя [секрета](../concepts/datamodel/secrets.md) с начальным IAM-токеном. Получить токен можно командой [`yc iam create-token`](https://yandex.cloud/ru/docs/cli/cli-ref/iam/cli-ref/create-token).

После успешного создания ресурса обновлять IAM-токен в секрете не требуется, дальнейшая аутентификация выполняется сервисным аккаунтом автоматически.
