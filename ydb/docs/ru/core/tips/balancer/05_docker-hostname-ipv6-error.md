# Ошибка `transport: Error while dialing: dial tcp: lookup [ipv6-address]` при запуске {{ ydb-short-name }} в Docker

Ошибка `transport: Error while dialing: dial tcp: lookup [ipv6-address]` возникает при попытке подключения клиента к {{ ydb-short-name }}, когда сервер возвращает недоступный IPv6-адрес в ответ на discovery/ListEndpoints. Это распространенная проблема при запуске {{ ydb-short-name }} через `docker run` без указания правильного hostname.

## Проблема

При запуске {{ ydb-short-name }} через Docker без указания опции `-h` (hostname), контейнер не знает свой публичный адрес или FQDN. В результате:

1. {{ ydb-short-name }} внутри контейнера определяет свой адрес как IPv6-адрес (например, `::1` или внутренний IPv6 контейнера)
2. Discovery-сервис возвращает этот IPv6-адрес клиентам
3. Клиенты пытаются подключиться по IPv6-адресу, который недоступен извне контейнера
4. Возникает ошибка подключения: `transport: Error while dialing: dial tcp: lookup [ipv6-address]`

### Симптомы
- Ошибка подключения при попытке соединения с {{ ydb-short-name }}
- В логах клиента видна ошибка с IPv6-адресом
- Discovery возвращает недоступные адреса узлов
- Подключение работает только изнутри контейнера

### Когда возникает
- При запуске {{ ydb-short-name }} через `docker run` без опции `-h`
- В окружениях, где IPv6 не настроен или недоступен
- При использовании Docker без явного указания hostname

## Решение

Указать правильный `hostname` при запуске контейнера {{ ydb-short-name }} через опцию `-h`

### Альтернативные варианты

Если `localhost` не подходит для вашего сценария, можно указать:
- Доменное имя машины: `-h my-server.example.com`
- IP-адрес машины: `-h 192.168.1.100`
- Любой другой доступный адрес, по которому можно подключиться к контейнеру

## Как это работает

При указании опции `-h`:
1. {{ ydb-short-name }} внутри контейнера узнает свой публичный адрес
2. Discovery-сервис возвращает клиентам корректные, доступные адреса
3. Клиенты успешно подключаются по указанному адресу
4. Ошибки подключения исчезают

## Рекомендации

1. **Всегда указывайте `-h` при запуске {{ ydb-short-name }} в Docker** - это предотвращает проблемы с IPv6
2. **Используйте понятные hostname** - `localhost` для локальной разработки, доменные имена для продакшена
3. **Проверяйте endpoints после запуска** - убедитесь, что discovery возвращает доступные адреса
4. **Документируйте настройки запуска** - чтобы другие разработчики не столкнулись с той же проблемой

Следование этим рекомендациям избавит от ошибок подключения и обеспечит стабильную работу {{ ydb-short-name }} в Docker-окружении.
