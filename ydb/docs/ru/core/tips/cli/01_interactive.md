# Эффективная работа в интерактивном режиме CLI

## Проблема

Многие разработчики используют {{ ydb-short-name }} CLI только для выполнения отдельных команд, не зная о мощном интерактивном режиме, который значительно упрощает работу с базой данных. Вместо удобного интерактивного сеанса они постоянно перезапускают CLI с новыми параметрами, теряют историю запросов и лишаются полезных функций автодополнения.

**Пример плохого подхода:**
```bash
# Постоянный перезапуск CLI для каждого запроса
ya ydb -e grpcs://localhost:2135 -d /local sql "SELECT * FROM users LIMIT 10"
ya ydb -e grpcs://localhost:2135 -d /local sql "SELECT COUNT(*) FROM users"
ya ydb -e grpcs://localhost:2135 -d /local sql "DESCRIBE TABLE users"
```

## Решение

Используйте интерактивный режим {{ ydb-short-name }} CLI для эффективной работы с базой данных. Этот режим предоставляет полноценную среду для выполнения запросов с историей команд, автодополнением и подсветкой синтаксиса.

### Запуск интерактивного режима

```bash
# Используя профиль
ya ydb --profile my-profile

# Или с явными параметрами подключения
ya ydb -e grpcs://localhost:2135 -d /local
```

### Основные возможности

#### 1. История запросов
- Используйте стрелки вверх/вниз для навигации по истории
- Поиск в истории: `Ctrl + R`
- Сохранение истории между сессиями

#### 2. Автодополнение и подсказки
- `Tab` - автодополнение ключевых слов и имен таблиц
- Интерактивные подсказки при вводе
- Поддержка контекстного автодополнения

#### 3. Подсветка синтаксиса
Цветовое выделение ключевых слов YQL, строк, чисел и комментариев для лучшего восприятия структуры запросов.

#### 4. Горячие клавиши
- Стрелки - навигация по истории
- `Ctrl + R` - поиск в истории
- `Ctrl + D` - выход из режима

### Пример эффективного использования

**Хороший подход:**
```bash
# Запускаем интерактивный сеанс
ya ydb --profile production

# В интерактивном режиме:
SELECT * FROM users LIMIT 10;
SELECT COUNT(*) FROM users;
DESCRIBE TABLE users;

-- Используем историю для повторного выполнения
-- (стрелка вверх для предыдущего запроса)
```

### Преимущества интерактивного режима

1. **Экономия времени** - не нужно постоянно перезапускать CLI
2. **Удобство отладки** - быстрый доступ к истории запросов
3. **Повышение продуктивности** - автодополнение и подсветка синтаксиса

### Рекомендации

- Настройте профиль подключения для быстрого доступа
- Используйте историю запросов для повторного выполнения сложных запросов

Интерактивный режим превращает {{ ydb-short-name }} CLI из простого инструмента выполнения команд в мощную среду разработки и администрирования баз данных.
