# Использование профилей подключения для удобства работы с {{ ydb-short-name }} CLI

## Проблема

При работе с {{ ydb-short-name }} CLI разработчики часто сталкиваются с необходимостью многократно указывать одни и те же параметры подключения:
- Эндпоинт базы данных
- Путь к базе данных
- Параметры аутентификации (токены, ключи)
- Дополнительные настройки подключения

**Пример плохого подхода** - указание всех параметров в каждой команде:

```bash
# ❌ Плохо: повторение одних и тех же параметров
ya ydb \
  -e grpcs://ydb.example.com:2135 \
  -d /ru-central1/b1g8skpblkos03m2lu8m/etn02q4l3bov9v7k5g4k \
  --yc-token-file ~/.ydb/token \
  scheme ls

ya ydb \
  -e grpcs://ydb.example.com:2135 \
  -d /ru-central1/b1g8skpblkos03m2lu8m/etn02q4l3bov9v7k5g4k \
  --yc-token-file ~/.ydb/token \
  discovery list
```

Это приводит к:
- **Ошибкам** при копировании длинных команд
- **Неудобству** при частом использовании CLI
- **Проблемам безопасности** при хранении чувствительных данных в истории команд

## Решение

{{ ydb-short-name }} CLI предоставляет механизм **профилей подключения** - именованных наборов параметров соединения, сохраненных в конфигурационном файле.

### Создание профиля

**Хороший подход** - использование интерактивного режима создания профиля:

```bash
# ✅ Хорошо: создание профиля один раз
ya ydb init
```

В интерактивном режиме CLI запросит все необходимые параметры:
- Имя профиля
- Эндпоинт базы данных
- Путь к базе данных
- Метод аутентификации и соответствующие параметры

Также можно создать профиль одной командой `ya ydb config profile create`, без интерактива.

**Пример создания профиля для облачной БД через Yandex Cloud CLI:**

```bash
# ✅ Отличный подход: автоматическое создание профиля из облачной конфигурации
yc ydb database get my-database | ya ydb config profile create my-database --stdin
```

### Использование профиля

После создания профиля команды становятся значительно короче и безопаснее:

```bash
# ✅ Хорошо: использование профиля
ya ydb -p my-database scheme ls
ya ydb -p my-database discovery list
ya ydb -p my-database sql -s "SELECT * FROM my_table"
```

### Активация профиля по умолчанию

Для максимального удобства можно активировать профиль, который будет использоваться автоматически:

```bash
# ✅ Отличный подход: активация профиля по умолчанию
ya ydb config profile activate my-database

# Теперь команды работают без указания профиля
ya ydb scheme ls
ya ydb discovery whoami
```

### Примеры конфигурационных файлов

**Хорошая конфигурация профиля** (файл `~/.ydb/config/config.yaml`):

```yaml
profile_my-database:
  endpoint: grpcs://ydb.example.com:2135
  database: /ru-central1/b1g8skpblkos03m2lu8m/etn02q4l3bov9v7k5g4k
  authentication:
    method: sa-key-file
    data: ~/my_sa_key_file
  iam_endpoint: iam.api.cloud.yandex.net
```

**Плохая практика** - хранение чувствительных данных в открытом виде:

```yaml
# ❌ Плохо: токен в открытом виде
profile_unsafe:
  endpoint: grpcs://ydb.example.com:2135
  database: /my/database
  auth_method: token
  token: "секретный_токен_в_открытом_виде"  # Опасно!
```

### Управление несколькими профилями

Для работы с разными окружениями создавайте отдельные профили:

```bash
# Профили для разных окружений
ya ydb config profile create production
ya ydb config profile create staging
ya ydb config profile create development

# Быстрое переключение между окружениями
ya ydb -p production scheme ls    # Продакшен
ya ydb -p staging scheme describe  # Стейжинг
ya ydb -p development sql -s "..." # Разработка
```

### Просмотр и управление профилями

```bash
# Список всех профилей
ya ydb config profile list

# Просмотр параметров конкретного профиля
ya ydb config profile get my-database

# Удаление профиля
ya ydb config profile delete old-profile
```

## Лучшие практики

1. **Используйте осмысленные имена профилей** - `production`, `staging`, `dev-backup`
2. **Храните токены в файлах**, а не в конфигурации профиля
3. **Регулярно обновляйте токены** в файлах аутентификации
4. **Используйте активированный профиль** для повседневной работы

## Распространенные ошибки и их решение

**Ошибка**: "More than one auth method were provided via options"
```bash
# ❌ Проблема: конфликт методов аутентификации
ya ydb --use-metadata-credentials --yc-token-file ~/.ydb/token scheme ls
```
**Решение**: Используйте только один метод аутентификации в профиле

**Ошибка**: Неправильный путь к конфигурационному файлу
- Внутренняя сборка {{ ydb-short-name }} CLI (`ya ydb`): `~/.ydb/config/config.yaml` (с точкой перед ydb)
- Публичная сборка {{ ydb-short-name }} CLI: `~/ydb/config/config.yaml`

**Ошибка**: Конфликт параметров командной строки и профиля
```bash
# ❌ Проблема: параметры из командной строки переопределяют профиль
ya ydb -p my-database -e другой-эндпоинт scheme ls
```
**Решение**: Параметры командной строки имеют приоритет над профилем - будьте осторожны

## Расширенные возможности

### Копирование профилей между системами

```bash
# Экспорт профиля в файл
ya ydb config profile get my-profile > profile-backup.yaml

# Импорт профиля на другой машине
ya ydb config profile create new-profile < profile-backup.yaml
```

### Использование переменных окружения

{{ ydb-short-name }} CLI также поддерживает переменные окружения для параметров подключения:

```bash
# Установка переменных окружения
export YDB_ENDPOINT=grpcs://ydb.example.com:2135
export YDB_DATABASE=/my/database
export YDB_TOKEN_FILE=~/.ydb/token

# Теперь команды работают без профиля
ya ydb scheme ls
```

## Заключение

Профили подключения {{ ydb-short-name }} CLI значительно упрощают работу с базой данных, устраняя необходимость повторного ввода параметров подключения. Правильное использование профилей повышает производительность, снижает вероятность ошибок и улучшает безопасность работы с чувствительными данными.

Ключевые преимущества профилей:
- **Экономия времени** - один раз настроил, используешь везде
- **Безопасность** - токены хранятся в защищенных файлах
- **Гибкость** - быстрое переключение между окружениями
- **Автоматизация** - удобство использования в скриптах

Начните использовать профили сегодня и почувствуйте разницу в удобстве работы с {{ ydb-short-name }} CLI!
