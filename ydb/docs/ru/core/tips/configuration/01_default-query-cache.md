# Оптимизация кэша запросов под требования приложения

## Проблема

Дефолтные серверные настройки кэша запросов в {{ ydb-short-name }} (500 мегабайт?) могут быть неоптимальными для конкретного использования приложения. Это приводит к быстрому вымыванию кэша запросов и увеличению нагрузки на CPU из-за повторных компиляций одинаковых запросов.

## Решение

Перенастроить размера кэша на узел {{ ydb-short-name }} <конфиг/CMS/UI>
