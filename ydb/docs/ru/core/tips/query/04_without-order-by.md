# Порядок результатов с и без ORDER BY

## Проблема

В {{ ydb-short-name }} строковые таблицы всегда упорядочены по первичному ключу, но это не означает, что результаты SELECT-запросов будут возвращаться в определенном порядке, если явно не указан `ORDER BY`. 

Без использования `ORDER BY`:
- Порядок результатов может меняться между выполнениями одного и того же запроса
- Порядок зависит от внутренней реализации {{ ydb-short-name }} и может изменяться при:
  - Изменении схемы таблицы
  - Разделении или объединении партиций
  - Изменении версии {{ ydb-short-name }}
  - Параллельном выполнении запроса на разных партициях
- Приложение не может полагаться на порядок результатов, что может привести к:
  - Непредсказуемому поведению пользовательского интерфейса
  - Ошибкам в бизнес-логике, зависящей от порядка
  - Проблемам с тестированием и воспроизводимостью результатов

## Решение

Всегда используйте `ORDER BY` для запросов, где порядок результатов важен. Это гарантирует:
- Предсказуемый и стабильный порядок результатов
- Воспроизводимость результатов между выполнениями
- Корректную работу приложения, зависящего от порядка данных

**Важно:**
- `ORDER BY` может влиять на производительность, особенно при сортировке больших объемов данных
- Для эффективной сортировки используйте индексы на полях, по которым выполняется сортировка
- При сортировке по первичному ключу или его префиксу производительность будет оптимальной
- Избегайте сортировки без `LIMIT` на больших таблицах


{% list tabs %}

- YQL

  {% cut "Плохой пример" %}
  ```yql
-- Плохой пример: запрос без ORDER BY
-- Порядок результатов непредсказуем
SELECT id, name, created_at FROM users
WHERE status = 'active'
LIMIT 10;
-- Порядок может меняться между выполнениями!
  ```
  {% endcut %}

  {% cut "Хороший пример" %}
  ```yql
-- Хороший пример: использование ORDER BY
-- Порядок результатов гарантирован и предсказуем
SELECT id, name, created_at FROM users
WHERE status = 'active'
ORDER BY created_at DESC
LIMIT 10;
-- Результаты всегда отсортированы по дате создания
  ```
  {% endcut %}

  ```go
result, err := session.Execute(ctx, `

`)

// Приложение не должно полагаться на порядок
  ```
  {% endcut %}

  {% cut "Хороший пример" %}
  ```go
// Хороший пример: явное указание ORDER BY
result, err := session.Execute(ctx, `

    LIMIT 10
`)
// Порядок результатов гарантирован
  ```
  {% endcut %}


  {% cut "Хороший пример" %}

  ```python
    "LIMIT 10"

# Порядок гарантирован
users = [row for row in result]
  ```
