# `BulkUpsert` для таблиц с синхронными индексами

## Проблема

Метод `BulkUpsert` не поддерживается для таблиц с синхронными вторичными индексами из-за архитектурных ограничений. Синхронные индексы требуют выполнения распределённых транзакций для поддержания строгой согласованности, что несовместимо с механизмом `BulkUpsert`.

## Решение

### 1. Использование UPSERT с параметризированным вводом

Для таблиц с синхронными индексами рекомендуется использовать оператор `UPSERT` с параметром `AS_TABLE($input)`. Этот подход поддерживает пакетную вставку данных и совместим с синхронными индексами.

**Пример YQL запроса:**
```sql
UPSERT INTO my_table
SELECT * FROM AS_TABLE($input);
```

**Пример с явным указанием колонок:**
```sql
UPSERT INTO my_table (pk_column, data_column1, data_column2)
SELECT pk_column, data_column1, data_column2 FROM AS_TABLE($input);
```

### 2. Стратегия загрузки данных с последующим созданием индексов

Если синхронные индексы не требуются на этапе загрузки данных, можно использовать следующий подход:

1. Создать таблицу без синхронных индексов
2. Загрузить данные с помощью `BulkUpsert` (оптимально для больших объемов)
3. Создать синхронные индексы после завершения загрузки

### 3. Использование асинхронных индексов

Если требования к согласованности позволяют использовать конечную согласованность вместо строгой, можно рассмотреть асинхронные индексы. Они совместимы с `BulkUpsert`, но обеспечивают конечную согласованность.

### Рекомендации по производительности

- **Размер транзакций**: Оптимальный размер - до 1 МБ данных или 100000 строк на транзакцию
- **Сортировка данных**: Сортируйте данные по первичному ключу для минимизации количества затронутых шардов
- **Шардирование**: Создавайте таблицы с достаточным количеством шардов (1 ГБ данных на шард)

### Когда использовать каждый подход

- **UPSERT с AS_TABLE**: Когда требуется строгая согласованность и поддержка синхронных индексов
- **BulkUpsert с последующим созданием индексов**: Для первоначальной загрузки больших объемов данных
- **Асинхронные индексы**: Когда допустима конечная согласованность и требуется максимальная производительность
