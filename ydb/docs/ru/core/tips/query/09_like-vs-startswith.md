# Оптимизация поиска по префиксу с StartsWith

## Проблема

При поиске строк по префиксу разработчики часто используют оператор `LIKE` с шаблоном, начинающимся с конкретного значения (например, `LIKE 'prefix%'`). Однако в {{ ydb-short-name }} такой подход может привести к следующим проблемам:

- **Неэффективное использование индексов** — оптимизатор {{ ydb-short-name }} может не использовать индекс для запросов с `LIKE`, даже если поиск идет по префиксу
- **FullScan** — запрос может выполнить полное сканирование таблицы вместо использования индекса
- **Высокая latency** — при больших объемах данных запросы с `LIKE` выполняются медленно
- **Избыточное потребление ресурсов** — полное сканирование требует больше CPU и памяти

Оператор `LIKE` предназначен для более сложных паттернов поиска (с wildcards в середине или конце строки), но для простого поиска по префиксу существуют более эффективные альтернативы.

## Решение

Используйте функцию `StartsWith` или оператор сравнения строк для поиска по префиксу. Это позволяет оптимизатору {{ ydb-short-name }} эффективно использовать индексы и избежать FullScan.

**Преимущества StartsWith:**
- Оптимизатор может использовать индекс для быстрого поиска
- Избегается полное сканирование таблицы
- Снижается latency запросов
- Уменьшается потребление ресурсов

**Когда использовать:**
- Поиск по префиксу строки (например, поиск пользователей по началу email)
- Фильтрация по началу текстового поля
- Поиск по кодам или идентификаторам с общим префиксом

**Когда использовать LIKE:**
- Поиск с wildcards в середине строки (например, `LIKE '%middle%'`)
- Сложные паттерны поиска
- Поиск с несколькими wildcards
