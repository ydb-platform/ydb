# Флаг `KeepInCache` для Table Service

## Проблема

При использовании устаревшего **Table Service** кэширование скомпилированных планов запросов по умолчанию **отключено**. Без явной установки флага `KeepInCache = true` каждый запрос компилируется заново, даже если его структура не изменилась.

Это приводит к:
- Избыточной нагрузке на CPU сервера
- Увеличенной latency каждого запроса
- Снижению общей пропускной способности

**Важно:** {{ ydb-short-name }} кэширует только планы выполнения запросов, но не результаты. Каждый запрос всегда возвращает актуальные данные из базы.

## Решение

### Query Service (рекомендуется)

**Query Service** — современный API для работы с {{ ydb-short-name }}. Кэширование планов запросов **включено по умолчанию**, никаких дополнительных настроек не требуется.

Переход на Query Service решает проблему автоматически.

### Table Service (устаревший)

Если вы всё ещё используете Table Service, необходимо явно устанавливать флаг `KeepInCache = true` в настройках каждого запроса.

**Без флага:**
```
Запрос 1 → Компиляция → Выполнение
Запрос 2 (та же структура) → Компиляция → Выполнение  ❌
Запрос 3 (та же структура) → Компиляция → Выполнение  ❌
```

**С флагом KeepInCache = true:**
```
Запрос 1 → Компиляция → Кэш → Выполнение
Запрос 2 (та же структура) → Кэш → Выполнение  ✅
Запрос 3 (та же структура) → Кэш → Выполнение  ✅
```

## Важное условие

Кэширование планов эффективно работает только с **параметризованными запросами**. 

Если встраивать значения напрямую в текст запроса (через конкатенацию строк), каждая комбинация значений создаёт уникальный текст запроса, который требует отдельной компиляции. Флаг `KeepInCache` в этом случае не поможет.

Подробнее см. статью "Параметризованные запросы для производительности".


{% cut "Плохой пример" %}
```python
# Плохо: Table Service без флага KeepInCache
# Каждый запрос компилируется заново

import ydb

def get_user_table_bad(table_client, session, user_id):
    session.transaction().execute(
        "SELECT * FROM users WHERE user_id = $userId",
        {"$userId": user_id},
        # KeepInCache не установлен — план НЕ кэшируется!
    )
```
{% endcut %}

{% cut "Хороший пример" %}
```python
import ydb

# Хорошо: Table Service с флагом KeepInCache
# План кэшируется после первой компиляции
def get_user_table_good(table_client, session, user_id):
    session.transaction().execute(
        "SELECT * FROM users WHERE user_id = $userId",
        {"$userId": user_id},
        settings=ydb.ExecDataQuerySettings().with_keep_in_cache(True)
    )

# Лучше: Query Service — кэширование включено по умолчанию
# Никаких дополнительных настроек не требуется
def get_user_query(pool: ydb.QuerySessionPool, user_id):
    pool.execute_with_retries(
        "SELECT * FROM users WHERE user_id = $userId",
        {"$userId": user_id}
    )
```
{% endcut %}
