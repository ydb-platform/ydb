# Ускорение запросов с помощью вторичных индексов

## Проблема

При выполнении запросов по полям, которые не входят в первичный ключ, {{ ydb-short-name }} вынуждена выполнять полное сканирование таблицы (FullScan). Это означает, что система должна прочитать все строки таблицы и проверить условие для каждой из них. При росте объема данных FullScan становится крайне медленным и ресурсоемким, что приводит к:

- Высокой latency запросов (запросы могут выполняться секундами или минутами)
- Избыточному потреблению CPU и памяти
- Снижению пропускной способности системы
- Ухудшению пользовательского опыта

Индекс — это структура данных, которая увеличивает скорость операций извлечения данных на основе одного или нескольких столбцов. Это аналог индекса в книге: вместо того чтобы просматривать каждую страницу книги, чтобы найти нужный раздел, вы можете обратиться к индексу в конце книги и быстро перейти к нужной странице.

## Решение

Создание вторичных индексов на часто используемых полях позволяет избежать FullScan и значительно снизить latency запросов. Когда выполняется запрос на основе столбца (или столбцов), по которому (которым) создан индекс, {{ ydb-short-name }} может использовать этот индекс, чтобы быстро найти соответствующие строки, минуя полный перебор всех данных.

**Рекомендации:**
- Создавайте индексы на столбцах, которые часто используются в условиях WHERE, JOIN и ORDER BY
- Учитывайте баланс между производительностью чтения и стоимостью поддержки индекса (индексы замедляют операции записи)
- Используйте составные индексы для запросов с несколькими условиями
- Рассмотрите использование covering index (покрывающего индекса) для запросов, которые читают только индексированные поля

- YQL

  {% cut "Плохой пример" %}
  ```yql
-- Плохой пример: запрос без индекса по полю email
-- YDB вынуждена выполнить FullScan всей таблицы
SELECT * FROM users WHERE email = 'user@example.com';
-- При большом объеме данных это будет очень медленно
  ```
  {% endcut %}

  {% cut "Хороший пример" %}
  ```yql
-- Альтернативный хороший пример: использование глобального индекса
CREATE TABLE users (
    id Uint64,
    email Utf8,
    name Utf8,
    PRIMARY KEY (id),
    INDEX idx_email GLOBAL ON (email)
);

-- Запрос автоматически использует индекс
SELECT * FROM users WHERE email = 'user@example.com';
  ```
  {% endcut %}

- Go

  {% cut "Плохой пример" %}
  ```go
// Плохой пример: запрос без индекса
// При большом объеме данных выполняется FullScan
result, err := session.Execute(ctx, `
    SELECT * FROM users WHERE email = $email
`, table.NewQueryParameters(
    table.ValueParam("$email", types.UTF8Value("user@example.com")),
))
// Медленно на больших таблицах!
  ```
  {% endcut %}

  {% cut "Хороший пример" %}
  ```go
// Хороший пример: запрос с использованием индекса
// Сначала создайте индекс:
// CREATE TABLE index_users_by_email (
//     email Utf8,
//     id Uint64,
//     PRIMARY KEY (email)
// );

result, err := session.Execute(ctx, `
    SELECT u.* FROM users AS u VIEW idx_users_by_email
    WHERE email = $email
`, table.NewQueryParameters(
    table.ValueParam("$email", types.UTF8Value("user@example.com")),
))
// Быстро благодаря использованию индекса
  ```
