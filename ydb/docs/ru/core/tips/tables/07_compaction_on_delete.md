# Настройка компакшена для эффективного удаления данных

## Проблема

В {{ ydb-short-name }} удаление записей технически производится добавлением "могильного камня" (tombstone), а физическая очистка от мусора выполняется компактификатором. По умолчанию компактификатор работает с низкой частотой, что может привести к:

- Накоплению могильных камней и увеличению размера таблицы
- Ухудшению производительности чтения из-за необходимости сканирования удаленных записей
- Задержкам в освобождении дискового пространства
- Повышению нагрузки на систему при массовых удалениях

## Решение

Для эффективной очистки данных необходимо настроить параметры background compaction:

### Основные настройки компактификации

1. **Увеличение частоты компактификации**:
   - Настройка `MinOperationRepeatDelay` - уменьшение времени между повторными запусками компактификации
   - Увеличение `MaxRate` - максимальное количество операций компактификации в интервале `RoundInterval`

2. **Параллельная обработка**:
   - Увеличение `InflightLimit` - максимальное количество шардов, для которых одновременно выполняются операции компактификации
   - Настройка `WakeupInterval` и `MinWakeupInterval` для управления повторными попытками

3. **Приоритизация удалений**:
   - Использование очереди `QueueRowDeletes` для приоритетной обработки шардов с большим количеством удаленных записей

### Рекомендуемые значения

Для сценариев с интенсивным удалением данных рекомендуется:

```yaml
BackgroundCompaction:
  IsCircular: true
  InflightLimit: 10-20  # увеличить для параллельной обработки
  MinOperationRepeatDelay: 300s  # уменьшить интервал между запусками
  MaxRate: 100  # увеличить максимальную частоту операций
  RoundInterval: 3600s  # интервал подсчета частоты
```

### Мониторинг эффективности

Следите за метриками:
- Количество могильных камней в таблицах
- Размер данных до и после компактификации
- Загрузка CPU при выполнении компактификации
- Время выполнения операций удаления
