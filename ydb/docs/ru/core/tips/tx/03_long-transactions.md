# Оптимизация длительности транзакций

## Проблема

Длительные транзакции в {{ ydb-short-name }} приводят к увеличению вероятности возникновения конфликтов и ошибок `Transaction Locks Invalidated` (TLI), что может привести к деградации производительности системы. {{ ydb-short-name }} использует оптимистические блокировки и MVCC для управления конкурентным доступом, где конфликты часто обнаруживаются лишь на этапе коммита транзакции.

**Основные проблемы длительных транзакций:**
- Увеличение времени удержания блокировок
- Повышенная вероятность конфликтов с другими транзакциями
- Частые ошибки TLI, требующие повторного выполнения транзакций
- Снижение общей пропускной способности системы

## Решение

### 1. Избегайте сложных и длинных действий внутри интерактивной транзакции

### 2. Используйте флаг коммита для последнего запроса

**Плохой подход** - явный коммит:

```python
# ПЛОХО: Явный вызов tx.commit()
with session.transaction(tx_mode=tx_mode) as tx:
    for i, query in enumerate(queries):
        result = tx.execute(prepared_statement, commit_tx=False)  # Всегда false
    tx.commit()  # Дополнительное время на коммит
```

**Хороший подход** - автоматический коммит в последнем запросе:

```python
# ХОРОШО: Автоматический коммит в последнем запросе
with session.transaction(tx_mode=tx_mode) as tx:
    len_queries = len(queries)
    for i, query in enumerate(queries):
        # Передавать commit_tx=True только в последнем запросе
        # Сервер в таком случае сразу выполнил commit
        is_last_query = i >= len_queries - 1
        result = tx.execute(prepared_statement, commit_tx=is_last_query)
```

### 3. Меняйте данные в нескольких небольших транзакциях  вместо одной большой

### 4. Используйте ReadOnly транзакции для работы с данными которые не нужно менять

**ReadOnly** режимы не создают конфликтов и не ломают локи **ReadWrite** транзакций

### 5. Мониторинг и диагностика

Настройте мониторинг метрик TLI в Grafana:

1. Откройте панель **DB Overview**
2. Следите за графиком **Transaction Locks Invalidation**

## Рекомендации

1. **Минимизируйте время транзакции** - выполняйте долгую бизнес-логику вне транзакции
2. **Используйте подготовленные запросы** - для уменьшения времени компиляции
3. **Разделяйте данные** - для уменьшения конфликтов
4. **Выбирайте подходящий режим изоляции** - в зависимости от требований к консистентности
5. **Настройте мониторинг** - для своевременного обнаружения проблем

Чем короче транзакция, тем ниже вероятность возникновения ошибок TLI и выше общая производительность системы.
