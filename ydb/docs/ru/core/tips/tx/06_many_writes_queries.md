# Пишущие много строк read-write и write-only транзакции

## Проблема

Выполнение read-write и write-only транзакций, которые пишут большое число строк в строковые таблицы, может приводить к ошибке `Out of buffer memory`, т.к. {{ ydb-short-name }} в данный момент перед записью в строковые таблицы накапливает данные в памяти и только в конце отправляет их в шарды.

## Решение

### 1. Разделить транзакцию на несколько меньших транзакций.

Транзакции, пишущие в строковые таблицы, следует делать небольшими, чтобы объем записываемых данных не превышал размер буфера для записи (64МБ).

Также желательно, чтобы одна транзакция затрагивала только один шард. Это позволит избежать расходов на координацию распределенных транзакций.

### 2. Для заливки данных использовать пакетную вставку данных

Если транзакция представляет собой заливку данных в таблицу, то вместо нее лучше использовать специализированный и более эффективный метод SDK `BulkUpsert` (см. [{#T}](../../recipes/ydb-sdk/bulk-upsert.md)).
