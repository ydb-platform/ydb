* Индекс всегда [локальный](../../../../concepts/glossary.md#local-index) (`LOCAL`); [глобального](../../../../concepts/glossary.md#secondary-index) варианта нет.
* В запросах не используется синтаксис `VIEW <index>` (в отличие, например, от [полнотекстовых индексов](../../../../dev/fulltext-indexes.md)).
* Фильтр применяется при чтении только к тем фрагментам данных, для которых при записи или [слиянии порций](../../../../concepts/glossary.md#compaction) уже сохранён блок индекса, для остальных фрагментов пропуск по этому индексу не выполняется.
