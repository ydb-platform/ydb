Параметры читателя:

* `type` — тип читателя. Возможные значения: `STREAMING` и `SHARED`. Значение по умолчанию `STREAMING`.
* `important` — признак важного читателя. Данные из топика не будут удалены, пока их не обработают все важные читатели. Тип значения — `boolean`, значение по умолчанию — `false`.
* `availability_period` — определяет время доступности сообщений для читателя. Опция позволяет продлить время хранения сообщений в топике сверх [retention_period](#topic-parameters), вплоть до `availability_period`, если читатель не подтверждает их обработку. Тип значения — `Interval`. Несовместим с параметром `important`. Значение по умолчанию отсутствует.
* `read_from` — определяет момент записи сообщений, начиная с которого читатель будет получать данные. Сообщения, записанные ранее этого момента, читатель не получит. Тип значения — `Datetime`, `Timestamp` или `integer` (Unix timestamp в виде числа). Значение по умолчанию — `0` (чтение с самого раннего доступного в топике момента времени).

Параметры читателя, доступные только для разделяемого (общего) читателя:

* `keep_messages_order` — сохраняет порядок чтения сообщений. Если значение равно `true`, гарантируется порядок обработки сообщений в рамках одной группы сообщений. Значение по умолчанию `false`.
* `default_processing_timeout` — время обработки сообщения. Если за это время обработка сообщения не будет подтверждена и время обработки не будет продлено, сообщение вернется в очередь и будет передано на повторную обработку. Значение по умолчанию `Interval('PT30S')`.
* `max_processing_attempts` — максимальное количество попыток обработки одного сообщения (тип значения — `integer`). Опция поддерживается только при `dead_letter_policy = move` или `dead_letter_policy = delete`. Значение по умолчанию — `1000`.
* `dead_letter_policy` — действие, которое нужно выполнить с сообщением, если все попытки его обработки завершились неудачей (тип значения — `String`). Возможные значения: `delete`, `move`, `none`. Значение по умолчанию — `none`.
* `dead_letter_queue` — имя топика DLQ (тип значения — `String`). Обязателен при `dead_letter_policy = move` и не поддерживается при `dead_letter_policy = none`/`delete`.
