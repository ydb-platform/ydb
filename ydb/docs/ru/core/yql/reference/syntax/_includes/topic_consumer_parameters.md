Параметры читателя:

* `type` — тип читателя. Возможные значения: `STREAMING` и `SHARED`.
* `important` — признак важного читателя. Данные из топика не будут удалены, пока их не обработают все важные читатели. Тип значения — `boolean`, значение по умолчанию — `false`.
* `availability_period` — определяет время доступности сообщений для читателя. Опция позволяет продлить время хранения сообщений в топике сверх [retention_period](#topic-parameters), вплоть до `availability_period`, если читатель не подтверждает их обработку. Тип значения — `Interval`. Несовместим с параметром `important`. Значение по умолчанию отсутствует.
* `read_from` — определяет момент записи сообщений, начиная с которого читатель будет получать данные. Сообщения, записанные ранее этого момента, читатель не получит. Тип значения — `Datetime`, `Timestamp` или `integer` (Unix timestamp в виде числа). Значение по умолчанию — `0` (чтение с самого раннего доступного в топике момента времени).

Параметры читателя, доступные только для shared-читателя:

* `keep_messages_order` — сохраняет порядок чтения сообщений. Если значение равно `true`, гарантируется порядок обработки сообщений в рамках одной группы сообщений. Значение по умолчанию `false`.
* `default_processing_timeout` — время обработки сообщения. Если за это время обработка сообщения не будет подтверждена и время обработки не будет продлено, сообщение вернется в очередь и будет передано на повторную обработку. Значение по умолчанию 30 сек.
* `max_processing_attempts` — максимальное количество попыток обработки одного сообщения. Если это число будет превышено, сообщение будет удалено или перемещено в DLQ в зависимости от значения `dead_letter_policy`.  Значение по умолчанию 1000.
* `dead_letter_policy` — задает действие, которое нужно выполнить с сообщением, если все попытки его обработки завершились неудачей. Возможные значения: `delete`, `move`, `none`.  Значение по умолчанию `none`.
* `dead_letter_queue` — задает имя топика, в который будет перемещено сообщение, если превышено максимальное количество попыток обработки и задано `dead_letter_policy = move`. Значение по умолчанию отсутствует.
