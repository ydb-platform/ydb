# Запуск принудительного компакшена таблицы

При помощи команды `ALTER TABLE ... COMPACT` можно явно запустить компакшен данных таблицы, не дожидаясь срабатывания фонового компакшена. Команда полезна, например, в следующих сценариях:

* было выполнено массовое удаление данных и хочется быстрее освободить занимаемое место;
* были изменены настройки сжатия или [групп колонок](family.md), и хочется быстрее применить их к уже существующим данным.

```yql
ALTER TABLE table_name COMPACT [WITH (key = value [, ...])];
```

Команда создаёт фоновую операцию и синхронно ждёт её завершения. Ожидание можно прервать, операция продолжит выполнятся в фоне. Запущенный таким образом компакшен работает независимо от автоматического фонового компакшена.

## Параметры {#options}

В блоке `WITH` можно указать следующие параметры:

* `CASCADE` — компактить ли вместе с таблицей её [вторичные индексы](indexes.md). Допустимые значения: `TRUE` и `FALSE`. Значение по умолчанию — `FALSE` (компактится только сама таблица).
* `PARALLEL` — максимальное число партиций, которые могут одновременно компактиться в рамках этой операции. Допустимые значения — положительное целое число. Значение по умолчанию — 1. Общий лимит на одновременно работающие ручные компакшены для всей базы задаётся отдельно в конфигурации SchemeShard.

## Поведение {#behavior}

* Команда асинхронна: после её выполнения создаётся фоновая операция, и за её ходом можно следить с помощью команды [`ydb operation list compaction`](../../../../reference/ydb-cli/operation-list.md). Получить статус конкретной операции по её идентификатору можно командой [`ydb operation get`](../../../../reference/ydb-cli/operation-get.md), отменить — командой [`ydb operation cancel`](../../../../reference/ydb-cli/operation-cancel.md), удалить запись завершённой операции из списка — командой [`ydb operation forget`](../../../../reference/ydb-cli/operation-forget.md).
* Если для той же таблицы (либо для одного из её вторичных индексов при `CASCADE = TRUE`) уже выполняется операция компакшена, запуск завершится с ошибкой. Нужно либо дождаться завершения предыдущей операции, либо отменить её.
* Для запуска команды требуются те же права, что и для остальных действий `ALTER TABLE`. Параллельные изменения схемы таблицы при этом не блокируются.
* Принудительный компакшен также материализует заимствованные (CoW) данные, которые есть, например, у копий таблиц. В результате объём хранимых в базе данных может увеличиться.

## Примеры {#examples}

Запустить компакшен только основной таблицы:

```yql
ALTER TABLE series COMPACT;
```

Запустить компакшен таблицы вместе со всеми её вторичными индексами:

```yql
ALTER TABLE series COMPACT WITH (CASCADE = TRUE);
```

Запустить компакшен только impl-таблицы конкретного вторичного индекса:

```yql
ALTER TABLE `series/idx_release/indexImplTable` COMPACT;
```

Запустить компакшен таблицы, установив число одновременно компактящихся партиций:

```yql
ALTER TABLE series COMPACT WITH (PARALLEL = 5);
```
