# RESTORE

Выражение `RESTORE` восстанавливает таблицы из [коллекции резервных копий](../../../concepts/backup-collections.md).

```yql
RESTORE collection_name;
```

## Параметры

* `collection_name` — имя коллекции резервных копий для восстановления.

## Поведение восстановления

Операция восстановления:

* Восстанавливает все таблицы из последней резервной копии в коллекции
* Если существуют инкрементальные резервные копии, применяет их поверх последней полной резервной копии
* Восстанавливает таблицы в их состояние на момент резервной копии
* Перезаписывает существующие таблицы с теми же именами

{% note warning %}

Операция восстановления перезапишет любые существующие таблицы с такими же именами. Убедитесь, что вы сделали резервную копию или переименовали существующие таблицы перед восстановлением, если хотите их сохранить.

{% endnote %}

## Примеры

Восстановление из коллекции резервных копий:

```yql
-- Восстановить все таблицы из коллекции
RESTORE production_backups;
```

## Мониторинг операций восстановления

Операции восстановления выполняются асинхронно в фоновом режиме. Вы можете отслеживать их прогресс с помощью YDB CLI:

```bash
# Список операций восстановления
ydb operation list incbackup

# Получить детали операции
ydb operation get <operation-id>
```
