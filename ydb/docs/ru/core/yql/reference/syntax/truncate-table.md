
# TRUNCATE TABLE

Очищает все данные таблицы (включая индексы).

Если таблицы с таким именем не существует, возвращается ошибка.

{% note info %}

На данный момент `TRUNCATE TABLE` не поддерживается для таблиц, у которых:

1. Есть добавленный [поток изменений](alter_table/changefeed.md).
2. Есть [асинхронный вторичный индекс](../../../../concepts/secondary_indexes.md#async).

Обратите внимание, что некоторые из существующих концепций используют в своем фундаменте поток изменений. Например, [асинхронная репликация](../../../concepts/async-replication.md). Таким образом, `TRUNCATE TABLE` не поддерживается для таблиц, у которых есть асинхронная репликация.

В будущих версиях эти ограничения будут устранены.

{% endnote %}

{% note info %}

Операция `TRUNCATE TABLE` не является MVCC-safe: ее ***нельзя прервать или отменить*** после начала выполнения.

{% endnote %}


## Примеры

```yql
TRUNCATE TABLE my_table;
```
