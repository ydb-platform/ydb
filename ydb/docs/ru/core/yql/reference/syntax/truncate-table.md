
# TRUNCATE TABLE

Удаляет все пользовательские данные из указанной таблицы и ее индексов.

## Синтаксис

```yql
TRUNCATE TABLE <table_name>;
```

{% note info %}

По поводу операции `TRUNCATE TABLE` стоит сделать несколько оговорок:

1. Операцию нельзя прервать или отменить после начала выполнения.
2. Операцию нельзя выполнить, если у таблицы есть [асинхронный вторичный индекс](../../../concepts/secondary_indexes.md#async) или добавленный [поток изменений](alter_table/changefeed.md) (ограничения будут устранены в будущих версиях).

Обратите внимание, что некоторые из существующих концепций используют в своем фундаменте поток изменений. Примером такой концепции служит [асинхронная репликация](../../../concepts/async-replication.md). Таким образом, `TRUNCATE TABLE` не поддерживается для таблиц, у которых есть асинхронная репликация.

{% endnote %}

## Примеры

```yql
TRUNCATE TABLE `/Root/test/my_table`;
```

```yql
TRUNCATE TABLE my_table;
```
